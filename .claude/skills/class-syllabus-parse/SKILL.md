# シラバス解析スキル

`class-syllabus` スキルが生成したシラバスMarkdownを読み込み、後続スキル（`class-test`・`class-report` など）が利用するための構造化情報を抽出するスキル。

## なぜこのスキルが必要か

シラバスからの情報抽出ロジック（基本情報・スケジュール・到達目標・ルーブリック・名前正規化など）は、テスト作成・レポート作成・成績処理など複数のスキルで共通して必要になる。各スキルに同じ抽出ロジックを持たせると二重メンテが発生するため、解析処理をこのスキルに一元化する。

## 前提条件

- シラバスMarkdownファイルが `output/` 配下に存在すること（`/class-syllabus` スキルで事前に作成）
- 出力はコンテキスト表示のみ。ファイル書き出しは行わない

## 対話ルール

ユーザーへの質問は場面に応じて適切な方式を使い分ける:

- **自由入力が自然な場面**（ファイルパスの直接入力など）: プレーンテキストで質問する
- **選択肢が明確な場面**（シラバスファイルの選択、抽出結果の確認など）: `AskUserQuestion` ツールを使う。自由入力は自動付与される "Other" で対応される

不明点や曖昧な点がある場合は、推測で進めずにユーザーに確認すること。

## 処理フロー

```
Step 0: シラバスMarkdownの特定
Step 1: ファイル読み込み
Step 2: 基本情報の抽出・表示
Step 3: スケジュール（レポート・試験）の抽出・表示
Step 4: 到達目標・ルーブリックの抽出・表示
Step 5: 抽出結果全体の確認
```

### Step 0: シラバスMarkdownの特定

以下の優先順位でシラバスファイルを特定する:

1. **引数で渡された場合**: 引数のパスをそのまま使用する
2. **会話コンテキストに既にシラバスが読み込まれている場合**: `AskUserQuestion` で確認する
   - options: 「現在のコンテキストに読み込まれている『[科目名]』で進める」「別のシラバスを指定する」
3. **`output/` 配下にシラバスMarkdownが存在する場合**: `output/*/[年度]開講*.md` を glob でリストアップし、`AskUserQuestion` の options として提示する。3ファイル以下なら全件、それ以上なら代表的なものを出して "Other" で手動入力を受け付ける
4. **該当がなかった場合**: プレーンテキストで直接パスを質問する:
   ```
   シラバスMarkdownファイルのパスを教えてください
   （例: output/2026_アルゴリズム/26開講情報工学科3アルゴリズム.md）:
   ```

シラバスMarkdownが見つからない場合は `/class-syllabus` スキルでの事前作成を案内する。

### Step 1: ファイル読み込み

特定したファイルを Read ツールで読み込む。読み込み失敗時はエラー内容を提示し、Step 0 に戻る。

### Step 2: 基本情報の抽出・表示

以下の情報を抽出する:

- 科目名
- 担当教員
- 対象学年
- 開講期
- 単位
- 評価割合（評価方法ごとの配点比率）

表示フォーマット:
```
=== シラバス基本情報 ===
科目名: [科目名]
担当教員: [教員名]
対象学年: [学年]
開講期: [期]
単位: [単位]
評価割合:
  - 定期試験: [N]%
  - 小テスト: [N]%
  - レポート・課題: [N]%
  - ...
```

基本情報が見つからない場合、該当項目は「（記載なし）」と表示する。

### Step 3: スケジュールの抽出・表示

レポート・試験のスケジュールを一括で抽出する。抽出ロジックの詳細は `references/syllabus-extraction-guide.md` を参照。

表示フォーマット:
```
=== レポート・課題スケジュール ===
| レポート名 | 提出週 | テーマ |
|-----------|--------|-------|
| レポート1 | 第8週 | ... |
| ... | ... | ... |

=== 試験スケジュール ===
| 試験名 | 実施週 | 範囲 |
|--------|--------|------|
| 小テスト1 | 第5週 | 第1〜4週 |
| 期末試験 | 第15週 | 第1〜14週 |
| ... | ... | ... |
```

スケジュールに該当する記載がない場合は「（該当なし）」と表示する。両方とも抽出できない場合でも処理は継続する（後続スキルで必要に応じて手動入力を求める）。

### Step 4: 到達目標・ルーブリックの抽出・表示

表示フォーマット:
```
=== 到達目標 ===
[シラバスの到達目標をそのまま表示]

=== ルーブリック ===
| 評価観点 | 理想的 | 標準的 | 未到達 |
|----------|--------|--------|--------|
| ... | ... | ... | ... |
```

### Step 5: 抽出結果の確認

Step 2〜4 の全抽出結果をまとめて提示した後、`AskUserQuestion` で確認する:
- options: 「この内容で進む（後続スキルで使用する）」「抽出内容を修正したい」

「修正したい」が選ばれた場合、プレーンテキストで修正箇所を聞き、該当項目のみ差し替える。スケジュールの追加・削除もここで対応する。

確認完了後は、抽出結果がコンテキストに残っている状態で本スキルを終了する。後続スキル（`class-test`・`class-report` 等）はこの抽出結果を参照して動作する。

---

## 後続スキルからの呼び出し方

`class-test`・`class-report` などの後続スキルは、Step 0 で本スキルを呼ぶよう案内すること。呼び出し方の例:

```
このスキルは事前にシラバス情報を読み込む必要があります。
`/class-syllabus-parse` を実行してから、続きを進めてください。
```

本スキルを実行すると、以降のコンテキストに以下の情報が残る:
- シラバスの生Markdownテキスト
- Step 2〜4 で抽出・表示された構造化情報

後続スキルはこれらを再度 Read せず、コンテキストに残っている情報を参照する。
