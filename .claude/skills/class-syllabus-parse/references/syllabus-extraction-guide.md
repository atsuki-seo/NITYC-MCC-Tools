# シラバス抽出ガイド

シラバスMarkdownから基本情報・スケジュール・到達目標・ルーブリックを抽出する際の共通ルール。`class-syllabus-parse` スキルの Step 2〜4 で使用する。

## 抽出の基本方針

- **構造化テーブル優先**: `class-syllabus` スキルが生成する最新のシラバスには、スケジュールが構造化テーブル（`### レポート・課題スケジュール` / `### 試験スケジュール`）として出力される。まずこれを最優先で読み取る
- **フォールバック**: 構造化テーブルが無い既存シラバスに対しては、`## 授業の進め方` 本文のパターンマッチでフォールバック抽出する
- **補完**: `## 授業計画` 表の各週の `授業内容` 列は補完情報として使う
- **人間による確認前提**: 抽出結果は必ず Step 5 でユーザー確認を通す。自動抽出は「人間による確認前提の下書き生成」と位置づける

---

## 基本情報の抽出

シラバスMarkdown の冒頭セクション（`## 基本情報` または類似の見出し）から以下を抽出する:

| 項目 | 抽出方法 |
|------|---------|
| 科目名 | `教科名:` / `科目名:` / 文書タイトル（h1）から取得 |
| 担当教員 | `担当教員:` 行 |
| 対象学年 | `対象学年:` 行 |
| 開講期 | `開講期:` 行 |
| 単位 | `単位:` 行 |
| 評価割合 | `## 評価割合` 表、または `評価方法` 見出し下の表から取得 |

見つからない項目は「（記載なし）」として扱う（エラーにはしない）。

## レポート・課題スケジュールの抽出

### 優先パス: 構造化テーブルからの読み取り

`## 授業の進め方` の下位にある `### レポート・課題スケジュール` テーブルを最優先で読み取る。

```markdown
### レポート・課題スケジュール
| レポート名 | 提出週 | テーマ |
|-----------|--------|-------|
| レポート1 | 第8週 | 字句解析の設計とプログラミング実践 |
| レポート2 | 第11週 | 構文解析の設計とプログラミング実践 |
| 最終レポート | 第14週 | 独自プログラミング言語の設計と実装 |
```

前提:
- 列は `レポート名` / `提出週` / `テーマ` の固定3列
- `レポート名` は既に正規化済み
- `提出週` は `第[N]週` 形式
- 合計行や注釈行がテーブル内にある場合は除外する

### フォールバック: フリーテキストからのパターンマッチ

構造化テーブルが無い既存シラバスに対しては、`## 授業の進め方` 本文のパターンマッチで抽出する。

**積極パターン（採用する）**:

- `・レポート[丸数字または数字][:：][テーマ]（第[N]週提出）`
- `・[特殊名]レポート[:：][テーマ]（第[N]週提出）` — 特殊名: `中間`, `期末`, `最終`
- `・レポート[N]は第[N]週に提出`
- `レポート[丸数字または数字]（第[N]週）`

**否定パターン（採用しない）**:
- 「レポート返却」「レポート講評」「レポート解説」を含む行
- 「期限内に提出すること」「遅延提出は減点」など一般的な提出ルールの言及
- 「レポートにフィードバックを返す」などのメタ記述

### 補完: 授業計画からの補完

`## 授業計画` テーブルの `授業内容` 列に「レポート[N]作成」「レポート[N]提出」などの記述がある場合、取りこぼしたレポートや提出週の補完情報として使う。ただし `## 授業の進め方` で抽出した情報を優先する。

## 試験スケジュールの抽出

### 優先パス: 構造化テーブルからの読み取り

`## 授業の進め方` の下位にある `### 試験スケジュール` テーブルを最優先で読み取る。

```markdown
### 試験スケジュール
| 試験名 | 実施週 | 範囲 |
|--------|--------|------|
| 小テスト1 | 第5週 | 第1〜4週 |
| 小テスト2 | 第8週 | 第5〜7週 |
| 期末試験 | 第15週 | 第1〜14週 |
```

前提:
- 列は `試験名` / `実施週` / `範囲` の固定3列
- `試験名` は既に正規化済み（アラビア数字または特殊名）
- `実施週` は `第[N]週` 形式
- `範囲` は `第[X]〜[Y]週` または `全範囲` 形式

### フォールバック: フリーテキストからのパターンマッチ

構造化テーブルが無い既存シラバスに対しては、`## 授業の進め方` 本文のパターンマッチで抽出する。

**検出対象キーワード**:
- 「小テスト」「中間試験」「期末試験」「定期試験」「確認テスト」

**積極パターン（採用する）**:
- `・小テスト[丸数字または数字][:：]第[N]週（範囲: 第[X]〜[Y]週）`
- `・[中間|期末|定期]試験[:：]第[N]週（範囲: 第[X]〜[Y]週）`
- `小テスト[丸数字または数字]（第[N]週、範囲: 第[X]〜[Y]週）`
- `第[N]週に小テスト[丸数字または数字]（範囲: 第[X]〜[Y]週）`

**否定パターン（採用しない）**:
- 「試験の返却」「試験解説」「試験後のフィードバック」を含む行
- 「試験は実施しない」「試験は行わない」などの否定文
- 「試験対策として」「試験準備として」などのメタ記述

### 範囲記載の解釈

| シラバス表記 | 解釈 |
|-------------|------|
| `第X〜Y週` / `第X週〜第Y週` / `第X-Y週` | そのまま範囲として使用 |
| `全範囲` / `全内容` / `これまでの内容` | 第1週〜当該試験の前週 |
| `前半` / `後半` など曖昧な表現 | 抽出失敗扱いにする |

抽出失敗時は、試験名と実施週だけ抽出し、範囲は「（要確認）」として出力する。後続スキルで教員に確認を求める。

### 補完: 授業計画からの補完

`## 授業計画` テーブルの `授業内容` 列に `小テスト①（範囲: 第1〜4週）` のように記述がある場合、フォールバックで取りこぼした情報の補完に使う。`## 授業の進め方` で抽出した情報を優先する。

## 名前の正規化ルール

レポート名・試験名は、どの抽出パスでも最終的に以下の形式に統一する。この正規化した名前を、後続スキルがディレクトリ名・ファイル名・プレビュー表示で一貫して使用する。

### レポート名

| シラバス表記 | 正規化後 |
|-------------|---------|
| `レポート①`, `レポート1`, `第1レポート`, `第1回レポート` | `レポート1` |
| `レポート②`, `レポート2` | `レポート2` |
| `レポート③`, `レポート3` | `レポート3` |
| `課題A` | `レポート1`（英字→数字、出現順） |
| `課題B` | `レポート2` |
| `中間レポート` | `中間レポート` |
| `期末レポート` | `期末レポート` |
| `最終レポート` | `最終レポート` |

### 試験名

| シラバス表記 | 正規化後 |
|-------------|---------|
| `小テスト①`, `小テスト1`, `第1回小テスト` | `小テスト1` |
| `小テスト②`, `小テスト2` | `小テスト2` |
| `小テスト③`, `小テスト3` | `小テスト3` |
| `中間試験`, `中間テスト` | `中間試験` |
| `期末試験`, `期末テスト` | `期末試験` |
| `定期試験` | `期末試験`（期末試験として扱う、中間と区別される場合のみそのまま） |
| `確認テスト[N]` | `小テスト[N]` |

### 共通ルール

1. 丸数字（①②③…）・漢数字（一二三…）は半角アラビア数字に変換する
2. 複数の同種テスト・レポートが存在する場合は出現順で連番にする
3. 単独のテスト・レポート（「中間試験」「期末レポート」など）はシラバスの記載をそのまま使用する
4. 「第N回」表記は末尾の数字のみ取り出し、「小テスト1」「レポート1」形式に統一する
5. ファイルシステムで使えない文字（`/`, `:`, `\`, 空白連続）はハイフンに置換する

## 到達目標・ルーブリックの抽出

### 到達目標

`## 到達目標` 見出し以下の本文（段落または箇条書き）を取得する。形式は問わず、そのままユーザーに表示する。

### ルーブリック

`## ルーブリック` 見出し以下の表を取得する。標準フォーマット:

```markdown
| 評価観点 | 理想的 | 標準的 | 未到達 |
|----------|--------|--------|--------|
| ... | ... | ... | ... |
```

列数・列名が異なるテンプレートもあり得るため、表をそのまま読み取って保持する（列名の改変はしない）。

---

## 抽出結果のユーザー確認

`class-syllabus-parse` の Step 5 で、Step 2〜4 の抽出結果をまとめて提示する。`AskUserQuestion` で確認:

- options: 「この内容で進む（後続スキルで使用する）」「抽出内容を修正したい」

「修正したい」が選ばれた場合、プレーンテキストで修正箇所を聞き、該当項目のみ差し替える。スケジュールの追加・削除もここで対応する。

---

## 抽出が失敗した場合

以下のいずれかに該当する場合は、自動抽出を諦めて手動入力を求める:

- 基本情報（科目名・担当教員・対象学年）がすべて取得できない
- スケジュールの構造化テーブル・フォールバックの両方で記載を検出できず、かつ評価割合上はレポート・試験の配点がある

プレーンテキストで質問する:

```
シラバス情報を自動抽出できませんでした。以下の情報を教えてください:
- 科目名:
- レポート/試験の名前・実施週・範囲（該当するもの）:
- ...
```

複数項目がある場合は繰り返し入力を受け付ける。
