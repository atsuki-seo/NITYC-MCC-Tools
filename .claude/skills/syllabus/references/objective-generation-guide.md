# 到達目標生成ガイド

Step 2.5a で到達目標を生成・改善する際の手順と品質基準。

## 生成の入力情報

| 入力 | 必須/任意 | 用途 |
|------|-----------|------|
| 科目名 | 必須 | キーワード抽出、MCCカテゴリ推定 |
| 開設学科 | 必須 | レベル感の調整 |
| 対象学年 | 必須 | 難易度・到達レベルの調整 |
| MCCカテゴリ（指定時） | 任意 | 到達目標例の参照元 |
| 既存の到達目標（改善時） | 任意 | 改善のベース |

## 生成手順

1. **MCCカテゴリの特定**: 科目名からキーワードを抽出し、`mcc-mapping.md`（インデックス）のキーワードと照合して関連MCCカテゴリを特定する。ユーザーがMCCカテゴリを直接指定した場合はそれを使う
2. **到達目標例の参照**: 特定されたMCCカテゴリの詳細ファイル（`mcc-categories/` 配下）を読み、到達目標一覧から科目の内容に合致するものを選出する
3. **学年レベルに応じた難易度調整**:
   - 1〜2年生: 「説明できる」「理解できる」「記述できる」（知識・理解レベル）
   - 3〜4年生: 「設計できる」「実装できる」「活用できる」（応用・分析レベル）
   - 5年生・専攻科: 「評価できる」「提案できる」「判断できる」（評価・創造レベル）
4. **出力形式の選択**: 到達目標を生成する前に、記述形式を `AskUserQuestion` で確認する:
   - options: 「散文形式（段落文章）（推奨）」「箇条書き形式」
   - **散文形式（デフォルト）**: 3段構成（背景・前提 → 本講義の内容 → 目標）の段落文章にまとめる。作文例は `references/objective-examples.md` を参照
   - **箇条書き形式**: `references/teaching-method-guide.md` の箇条書きルール（最大3階層: `・` / `　・` / `　　・`）に従って記述する
5. **到達目標の生成**: 3〜5個程度を「〜できる」形式で生成し、手順4 で選択された形式で記述する
6. **確認**: 生成結果をプレーンテキストで表示し、`AskUserQuestion` で確認を取る

## 品質基準

実際のシラバスでの到達目標の記載例は `references/objective-examples.md` を参照。

- **測定可能な表現**: 「理解する」ではなく「説明できる」「実装できる」のように、達成の判定が可能な動詞を使う
- **ルーブリックへの分解可能性**: 各到達目標が理想的・標準的・未到達の3段階に自然に分解できる粒度にする
- **MCC到達目標例との整合性**: `mcc-categories/` 配下の該当カテゴリ詳細ファイルの到達目標一覧と矛盾しない内容にする
- **科目の範囲に収まる**: 当該科目で扱うべき範囲に限定し、前提科目や後続科目の内容を含めない

## 改善モード

既存の到達目標を改善する場合、以下の手順で進める。

### 出力形式

出力形式（散文形式・箇条書き形式）は SKILL.md の Step 2.5a で確認済みである。確認済みの形式で改善案を生成する（形式の詳細は「生成手順」の手順4-5を参照）。

### 改善観点

以下の観点でチェックし改善案を提示する:

- **曖昧な表現の具体化**: 「理解する」→「説明できる」、「学ぶ」→「実装できる」等
- **MCCカテゴリとの対応強化**: 関連するMCCカテゴリの到達目標例を参照し、不足しているトピックを提案
- **学年レベルとの整合性**: 対象学年に対して難易度が適切かチェック
- **粒度の調整**: 1つの到達目標に複数の独立した内容が含まれていないか確認

改善案は既存の到達目標との差分が分かるように表示する。

## MCCカテゴリ指定モード

ユーザーがMCCカテゴリを直接指定して生成する場合:

1. `mcc-mapping.md`（インデックス）のカテゴリ一覧を `AskUserQuestion`（multiSelect: true）で表示する
   - options: 関連しそうなカテゴリを先頭に配置（科目名から推定）
2. 選択されたカテゴリの詳細ファイル（`mcc-categories/` 配下）を読み、到達目標一覧とキーワードをベースに科目に合わせた到達目標を生成
3. Step 4（MCC対応付け）で、ここで指定されたカテゴリを自動適用する（ユーザーに確認の上）

## ルーブリック生成の参考

ルーブリック（理想的・標準的・未到達の3段階）の作文例は `references/rubric-examples.md` を参照。

## 確認フロー

生成した到達目標を選択された形式（散文形式または箇条書き形式）で表示した後、`AskUserQuestion` で確認する:
- options: 「この到達目標で確定する」「一部を修正したい」「再生成してほしい」

「一部を修正したい」→ プレーンテキストで修正箇所を聞き、反映後に再表示
「再生成してほしい」→ プレーンテキストで追加の指示や方向性を聞いてから再生成
