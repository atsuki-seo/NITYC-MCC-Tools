# 確定データJSONフォーマット

write_syllabus.py が期待する正確なJSON構造。キー名は以下の通りに正確に使用すること。

```json
{
  "教科書_教材": "https://moodle2026.center.yuge.ac.jp",
  "到達目標": "到達目標のテキスト",
  "ルーブリック": [
    {"評価観点": "観点1", "理想的": "理想的な到達レベル", "標準的": "標準的な到達レベル", "未到達": "未到達レベル"},
    {"評価観点": "観点2", "理想的": "...", "標準的": "...", "未到達": "..."}
  ],
  "教育方法等": {
    "概要": "生成した概要テキスト",
    "授業の進め方": "生成した授業の進め方テキスト",
    "注意点": "生成した注意点テキスト"
  },
  "授業計画": {
    "学期": "前期 or 後期",
    "週データ": [
      {"週番号": 1, "授業内容": "...", "到達目標": "..."},
      {"週番号": 2, "授業内容": "...", "到達目標": "..."}
    ]
  },
  "評価割合": {
    "観点別配分": [
      {
        "観点": "知識の基本的な理解",
        "配分": {"定期試験": 30, "小テスト": 10, "レポート": 0, "口頭発表": 0, "成果物実技": 0, "ポートフォリオ": 0, "その他": 0}
      },
      {
        "観点": "汎用的技能",
        "配分": {"定期試験": 20, "小テスト": 0, "レポート": 20, "口頭発表": 0, "成果物実技": 0, "ポートフォリオ": 0, "その他": 0}
      },
      {
        "観点": "主体的・継続的な学習意欲",
        "配分": {"定期試験": 0, "小テスト": 10, "レポート": 0, "口頭発表": 0, "成果物実技": 0, "ポートフォリオ": 0, "その他": 0}
      },
      {
        "観点": "態度・指向性",
        "配分": {"定期試験": 0, "小テスト": 0, "レポート": 0, "口頭発表": 0, "成果物実技": 0, "ポートフォリオ": 0, "その他": 10}
      }
    ]
  },
  "授業の属性": {
    "アクティブラーニング": "〇",
    "ICT利用": "〇",
    "遠隔授業対応": "〇"
  }
}
```

## 注意事項

- `週データ`（`週` ではない）を使用すること
- 評価割合の `配分` はネストされたオブジェクト構造にすること
- 観点名は正確に: 「知識の基本的な理解」「汎用的技能」「主体的・継続的な学習意欲」「態度・指向性」
- `ルーブリック` は配列で、各要素に `評価観点`・`理想的`・`標準的`・`未到達` の4キーを持つ
- `教科書_教材` はトップレベルのフィールド（`到達目標` と同階層）
- `授業の進め方` は `教育方法等` 内のフィールド
- 評価割合の全観点・全評価方法の配分値の合計は必ず100になること（上記サンプル: 30+10+20+20+10+10=100）
- Step 3 で「その他（出席点・授業態度等）」を選択した場合、JSONキーは `"その他"` を使用すること（括弧書きは含めない）
