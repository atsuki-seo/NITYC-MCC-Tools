# 確定データJSONフォーマット

write_syllabus.py が期待する正確なJSON構造。キー名は以下の通りに正確に使用すること。

```json
{
  "教科書_教材": "https://moodle2026.center.yuge.ac.jp",
  "到達目標": "到達目標のテキスト",
  "ルーブリック": [
    {"評価観点": "観点1", "理想的": "理想的な到達レベル", "標準的": "標準的な到達レベル", "未到達": "未到達レベル"},
    {"評価観点": "観点2", "理想的": "...", "標準的": "...", "未到達": "..."}
  ],
  "教育方法等": {
    "概要": "生成した概要テキスト",
    "授業の進め方": "生成した授業の進め方テキスト",
    "注意点": "生成した注意点テキスト"
  },
  "授業計画": {
    "学期": "前期 or 後期",
    "週データ": [
      {"週番号": 1, "授業内容": "...", "到達目標": "..."},
      {"週番号": 2, "授業内容": "...", "到達目標": "..."}
    ]
  },
  "評価割合": {
    "観点別配分": [
      {
        "観点": "知識の基本的な理解",
        "配分": {"定期試験": 30, "小テスト": 10, "レポート": 0, "口頭発表": 0, "成果物実技": 0, "ポートフォリオ": 0, "その他": 0}
      },
      {
        "観点": "汎用的技能",
        "配分": {"定期試験": 20, "小テスト": 0, "レポート": 20, "口頭発表": 0, "成果物実技": 0, "ポートフォリオ": 0, "その他": 0}
      },
      {
        "観点": "主体的・継続的な学習意欲",
        "配分": {"定期試験": 0, "小テスト": 10, "レポート": 0, "口頭発表": 0, "成果物実技": 0, "ポートフォリオ": 0, "その他": 0}
      },
      {
        "観点": "態度・指向性",
        "配分": {"定期試験": 0, "小テスト": 0, "レポート": 0, "口頭発表": 0, "成果物実技": 0, "ポートフォリオ": 0, "その他": 10}
```

## 注意事項

- `週データ`（`週` ではない）を使用すること
- 評価割合の `配分` はネストされたオブジェクト構造にすること
- **評価割合のキー名はテンプレート準拠**: `観点` の値と `配分` のキー名は、Excelテンプレートの評価割合シートから読み取った実際のヘッダー・観点名を使用すること。テンプレートによって名称が異なる場合がある（例: 「定期試験」→「試験」、「汎用的技能」→「思考・推論・創造への適応力」など）。Excel無しの場合は標準名を使用する
- 評価方法名・観点名は「テンプレートパターン」セクションを参照
- `ルーブリック` は配列で、各要素に `評価観点`・`理想的`・`標準的`・`未到達` の4キーを持つ
- `教科書_教材` はトップレベルのフィールド（`到達目標` と同階層）
- `授業の進め方` は `教育方法等` 内のフィールド
- 評価割合の全観点・全評価方法の配分値の合計は必ず100になること（上記サンプル: 30+10+20+20+10+10=100）
- Step 3 の選択肢名とJSONキー名の対応:
  - 「レポート・課題」→ JSONキーは `"レポート"`
  - 「成果物・実技」→ JSONキーは `"成果物実技"`
  - 「その他（出席点・授業態度等）」→ JSONキーは `"その他"`（括弧書きは含めない）
  - 上記はExcel無しでデフォルトパターン使用時の対応。Excel有りの場合はテンプレートのヘッダー名をそのまま使用する

## テンプレートパターン

テンプレートによって評価方法名・観点名が異なる。**Excel有りの場合**はテンプレートから動的に読み取った値を使用する。**Excel無しの場合**は以下の既知パターンからユーザーに選んでもらう（Step 3 で `AskUserQuestion` を使用）。

### 評価方法パターン

| パターン | 列1 | 列2 | 列3 | 列4 | 列5 | 列6 | 列7 |
|----------|-----|-----|-----|-----|-----|-----|-----|
| A（デフォルト） | 定期試験 | 小テスト | レポート | 口頭発表 | 成果物実技 | ポートフォリオ | その他 |
| B | 試験 | 発表 | 相互評価 | 態度 | ポートフォリオ | その他 | — |

### 観点パターン

| パターン | 観点1 | 観点2 | 観点3 | 観点4 |
|----------|-------|-------|-------|-------|
| A（デフォルト） | 知識の基本的な理解 | 汎用的技能 | 主体的・継続的な学習意欲 | 態度・指向性 |
| B | 知識の基本的な理解 | 汎用的技能 | 主体的・継続的な学習意欲 | 態度・指向性（人間力） |
| C | 知識の基本的能力 | 思考・推論・創造への適応力 | リーダーシップ・コミュニケーション力 | プレゼンテーション力 |
| D | 知識の基本的な理解 | 思考・推論・創造への適応力 | 主体的・継続的な学習意欲 | — |
| E | 知識の基本的な理解 | 思考・推論・創造への適応力 | リーダーシップ・コミュニケーション能力 | プレゼンテーション力 |

JSONの `配分` キー名とサンプルJSON中の `観点` の値は、選択されたパターンの名称を正確に使用すること。観点が「—」のパターンでは、その観点は `観点別配分` に含めない
