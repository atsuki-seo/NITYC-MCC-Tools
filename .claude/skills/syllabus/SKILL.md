# シラバス作成スキル

高専のシラバスをエクセルテンプレートとMCC（モデルコアカリキュラム）をもとに作成するスキル。ユーザーが入力した基本情報・到達目標・ルーブリックから、授業計画・評価割合・概要などを生成し、エクセルとMarkdownの両方に出力する。

## なぜこのスキルが必要か

高専のシラバス作成は、6つのシートにまたがるエクセルファイルへの記入が必要で、MCCとの対応付けも求められる。この作業は毎年繰り返されるが、到達目標やルーブリックから授業計画を導き出すプロセスは構造的で、支援の余地が大きい。

## 前提条件

- 1コマの授業時間は **90分** とする
- Python3 がインストールされていること
- 初回実行時に依存パッケージをインストールする:
  ```bash
  pip3 install -r ${CLAUDE_SKILL_DIR}/requirements.txt
  ```
- エクセルテンプレートは6シート構成（後述）

## 対話ルール

ユーザーへの質問は場面に応じて適切な方式を使い分ける:

- **自由入力が自然な場面**（教員名、科目名、ファイルパス、到達目標、ルーブリックなど）: プレーンテキストで質問する
- **選択肢が明確な場面**（定期試験の種類、MCC対応の有無、既存情報の利用、授業の進め方など）: `AskUserQuestion` ツールを使う。自由入力は自動付与される "Other" で対応される

不明点や曖昧な点がある場合は、推測で進めずにユーザーに確認すること。

## 処理フロー

```
Step 0: 教員名・科目名の取得
Step 1: Excelファイルの指定
  ├→ Excel有り → Step 2a: 読み取り → Step 2b: 既存情報の利用確認
  └→ Excel無し → Step 2c: 対話で全情報収集
Step 3: 授業構成パラメータ（回数・授業構成案・評価スタイル・試験・MCC）
Step 4: MCC対応付け（オプション）
Step 5: 生成（概要・授業計画・評価割合・属性）
Step 6: プレビュー・修正ループ
Step 7: Excel書き込み（Excel有りの場合）
Step 8: Markdown出力
Step 9: 完了レポート
```

### Step 0: 教員名・科目名の取得

最初に必ず教員名と科目名を確認する。これらはシラバスの基本であり、以降の全ステップで使用する。

プレーンテキストで質問する:
```
担当教員のお名前を教えてください:
科目名を教えてください:
```

### Step 1: Excelファイルの指定

プレーンテキストで質問する:
```
シラバスのエクセルテンプレートファイルのパスを教えてください
（ない場合は「なし」と入力してください）:
```

- パスが指定された場合 → Step 2a へ
- 「なし」の場合 → Step 2c へ

### Step 2a: Excel読み取り（Excel有りの場合）

同梱の読み取りスクリプトでデータを抽出する:

```bash
python3 ${CLAUDE_SKILL_DIR}/scripts/read_syllabus.py "<エクセルファイルパス>"
```

出力はJSON形式。スクリプトがエラーを返した場合（ファイル不在、シート不足など）はユーザーに伝えて対応を求める。

抽出した情報を整理して表示する:

```
=== シラバス基本情報 ===
教科名: [科目名]
科目番号: [番号]
科目区分: [区分]
授業形態: [形態]
単位: [単位]
開設学科: [学科]
対象学年: [学年]
開講期: [期]
週時間数: [時間]
担当教員: [教員名]

=== 到達目標 ===
（読み取った到達目標を表示）

=== ルーブリック ===
| 評価観点 | 理想的 | 標準的 | 未到達 |
|----------|--------|--------|--------|
| ... | ... | ... | ... |
```

**教員名・科目名の整合チェック**: Excel から読み取った値が Step 0 で入力された値と異なる場合、`AskUserQuestion` でどちらを使うか確認する:
- options: 「Step 0で入力した値を使う」「Excelの値を使う」

整合に問題がなければ、「この内容で合っていますか？修正があれば教えてください。」とプレーンテキストで確認する。

### Step 2b: 既存情報の利用確認（Excel有りの場合）

Excel に概要や授業計画が既に記入済みの場合、`AskUserQuestion` で確認する:

```
テンプレートに以下の既存情報が見つかりました:
- 概要: （記入済み / 空）
- 授業計画: （記入済み / 空）
- 評価割合: （記入済み / 空）
- 授業の属性: （記入済み / 空）
```

- options: 「既存情報をベースに修正・補完する」「既存情報を無視して新規作成する」

全て空の場合はこのステップをスキップする。

確認後 → Step 3 へ

### Step 2c: 対話で全情報収集（Excel無しの場合）

以下をプレーンテキストで順次収集する:

**1. 基本情報**
```
以下の基本情報を教えてください:
- 科目番号:
- 科目区分（例: 専門/必修）:
- 授業形態（例: 授業、演習）:
- 単位の種別と単位数（例: 履修単位: 1）:
- 開設学科:
- 対象学年:
- 開講期（例: 前期、後期、通年）:
- 週時間数:
- 教科書/教材:
```

**2. 到達目標**
```
この科目の到達目標を教えてください:
（複数ある場合は番号付きで入力してください）
```

**3. ルーブリック**

`AskUserQuestion` で方式を確認する:
- options: 「自分で入力する」「到達目標から自動生成してほしい」

「自分で入力する」の場合:
```
各評価観点について、理想的・標準的・未到達の基準を教えてください:

評価観点1:
- 理想的な到達レベル:
- 標準的な到達レベル:
- 未到達レベル:
```

「自動生成」の場合: 到達目標の内容からルーブリックを生成し、プレーンテキストで確認を取る。

収集完了後 → Step 3 へ

### Step 3: 授業構成パラメータの確認

以下を順に確認する（Excel有無にかかわらず必ず全項目を質問する）:

**授業回数**: プレーンテキストで質問する。
```
授業回数は何回ですか？（例: 15, 16, 30, 32）:
```

**授業構成案**: `AskUserQuestion` で授業スタイルを質問する。
- options: 「講義中心」「演習中心」「講義＋演習」「反転授業」

複合型（「講義＋演習」など講義と演習の両方を含むスタイル）が選ばれた場合、プレーンテキストで時間配分を質問する:
```
1コマ90分のうち、講義と演習の配分はどのくらいを想定していますか？
（例: 講義60分＋演習30分、講義45分＋演習45分）:
```

授業構成案と時間配分は、Step 5c の授業計画生成に反映する。各週の授業内容を記述する際に、構成に沿った流れ（例: 前半で概念を講義→後半で演習問題に取り組む）を意識して計画を組み立てる。

**使用する評価方法**: `AskUserQuestion`（multiSelect: true）で質問する。
- options: 「定期試験」「小テスト」「レポート・課題」「口頭発表」「成果物・実技」「ポートフォリオ」

選択された評価方法に応じて、以下のフォローアップ質問を行う:

- **定期試験**を選択 → `AskUserQuestion` で質問する:
  - options: 「中間試験のみ」「期末試験のみ」「中間・期末の両方」
- **小テスト**を選択 → `AskUserQuestion` で頻度を質問する:
  - options: 「毎週」「隔週」「単元ごと（3〜4回程度）」「中間・期末前のみ」
- **レポート・課題**を選択 → `AskUserQuestion` で頻度を質問する:
  - options: 「毎週」「隔週」「単元ごと（3〜4回程度）」「期末のみ」
- **口頭発表**を選択 → `AskUserQuestion` で形式を質問する:
  - options: 「個人発表」「グループ発表」「ディスカッション」
- **成果物・実技**を選択 → プレーンテキストで具体的な内容を質問する（自由度が高いため）
- **ポートフォリオ**を選択 → `AskUserQuestion` で形式を質問する:
  - options: 「学習記録ノート」「振り返りシート」「デジタルポートフォリオ」

**評価の重み付け方針**: `AskUserQuestion` で質問する。
- options: 「均等に配分」「特定の方法を重視したい」
- 「特定の方法を重視したい」を選んだ場合、プレーンテキストでどの方法をどの程度重視するか質問する

これらの回答は Step 5c（授業計画）と Step 5d（評価割合）に反映する。小テスト・レポートの頻度に応じて授業計画に実施タイミングを配置し、評価割合は選択した方法と重み付け方針に基づいて生成する。

**MCC対応**: `AskUserQuestion` で質問する。
- options: 「MCCとの対応付けを行う」「MCCとの対応付けは不要」

### Step 4: MCC対応付け（オプション）

Step 3 で MCC 対応を選択した場合のみ実施。

`references/mcc-mapping.md` を参照し、到達目標の内容からキーワードを抽出して関連するMCCカテゴリを特定する。

対応結果を表示する:

```
=== MCC対応分析 ===
到達目標1 → V-D-2 ソフトウェア（アルゴリズム、データ構造）
到達目標2 → V-D-1 プログラミング（制御構造、サブルーチン）
...

推奨追加項目:
- V-D-2: 計算量の比較・評価（現在の到達目標でカバー済み）
- ...
```

プレーンテキストで「この対応で問題ありませんか？修正があれば教えてください。」と確認する。

### Step 5: 授業計画・出力項目の生成

到達目標・ルーブリック・MCCの分析結果をもとに以下を生成する:

#### 5a. 概要（「教育方法等・自由使用枠」シート）

到達目標を要約し、科目の目的・扱う内容・期待される成果を自然な文章で記述する。到達目標の文言をそのまま並べるのではなく、科目全体の意義が伝わるように書く。

#### 5b. 注意点（「教育方法等・自由使用枠」シート）

「授業の進め方・方法」の内容から、学生への予習復習の推奨事項や評価上の注意を抽出・生成する。

#### 5c. 授業計画（「授業計画」シート）

到達目標とルーブリックを分解し、授業回数に応じて週ごとに配分する。

配分の考え方:
- 到達目標を達成するために必要なトピックを洗い出す
- トピックの依存関係を考慮して順序を決める（基礎→応用）
- 各週に「授業内容」と「週ごとの到達目標」を設定する
- 定期試験がある場合、該当週（中間: 8週目付近、期末: 最終週付近）に試験を設定する
- Step 3 で確認した授業構成案・時間配分を反映する。例えば「講義60分＋演習30分」なら、各週の授業内容に「前半: ○○の講義、後半: ○○の演習」のように構成を反映させる。演習中心なら演習課題の内容を具体的に記述する
- Step 3 で確認した小テスト・レポートの頻度に応じて、授業計画に実施タイミングを配置する（例: 小テスト「隔週」なら2週ごとに「小テスト」を記載、レポート「単元ごと」なら単元の区切りに「レポート提出」を記載）

#### 5d. 評価割合（「評価割合」シート）

Step 3 で選択された評価方法・頻度・重み付け方針をベースに、ルーブリックの評価観点と組み合わせて配分を生成する。

- 評価方法: 定期試験、小テスト、レポート、口頭発表、成果物実技、ポートフォリオ、その他
- 観点: 知識の基本的な理解、汎用的技能、主体的・継続的な学習意欲、態度・指向性
- 合計が100%になるようにする
- Step 3 でユーザーが選択しなかった評価方法は 0% とする
- 「均等に配分」なら選択された方法間で均等に分配し、「特定の方法を重視したい」ならユーザーの指定に沿って重み付けする
- 頻度が高い評価方法（例: 毎週の小テスト）は配分を大きめにするなど、頻度も考慮する

#### 5e. 授業の属性（「授業の属性・履修上の区分」シート）

授業方法の記述から以下を判定する:
- アクティブラーニング: 「反転授業」「グループワーク」「演習」「PBL」等 → 〇
- ICT利用: 「LMS」「Moodle」「オンライン」「プログラミング環境」等 → 〇
- 遠隔授業対応: 「オンライン対応」「ハイブリッド」等 → 〇

### Step 7: エクセルファイルへの書き込み（Excel有りの場合のみ）

`AskUserQuestion` で確認する:
- options: 「エクセルに書き込む」「Markdownのみ出力する（書き込みスキップ）」

「エクセルに書き込む」の場合:

書き込み前にバックアップを作成する:
```bash
cp "<元ファイル>" "<元ファイル>.bak"
```

書き込みスクリプトを使用する:
```bash
python3 ${CLAUDE_SKILL_DIR}/scripts/write_syllabus.py "<エクセルファイルパス>" '<確定データJSON>'
```

スクリプトの仕様:
- 既存のセル書式（フォント、罫線、セル結合など）を保持する
- 値の書き込みのみ行う
- 書き込み対象: 科目情報・到達目標シート（到達目標）、教育方法等シート（概要・注意点）、授業計画シート、評価割合シート、授業属性シート

**確定データJSONのフォーマット**（スクリプトが期待する正確な構造）:

```json
{
  "到達目標": "到達目標のテキスト",
  "教育方法等": {
    "概要": "生成した概要テキスト",
    "注意点": "生成した注意点テキスト"
  },
  "授業計画": {
    "学期": "前期 or 後期",
    "週データ": [
      {"週番号": 1, "授業内容": "...", "到達目標": "..."},
      {"週番号": 2, "授業内容": "...", "到達目標": "..."}
    ]
  },
  "評価割合": {
    "観点別配分": [
      {
        "観点": "知識の基本的な理解",
        "配分": {"定期試験": 0, "小テスト": 30, "レポート": 20, "口頭発表": 0, "成果物実技": 0, "ポートフォリオ": 0, "その他": 0}
      }
    ]
  },
  "授業の属性": {
    "アクティブラーニング": "〇",
    "ICT利用": "〇",
    "遠隔授業対応": "〇"
  }
}
```

注意: キー名は上記の通りに正確に使用すること。特に `週データ`（`週` ではない）と、評価割合の `配分`（ネストされたオブジェクト）に注意。

Excel無しの場合はこのステップをスキップする。

### Step 8: Markdown出力

プレーンテキストで保存先を確認する:
```
Markdownファイルの保存先を教えてください:
（デフォルト: エクセルファイルと同じディレクトリ / カレントディレクトリ）
ファイル名: [教科名].md
```

Step 6 のプレビュー内容をそのままMarkdownファイルとして保存する。出力するセクション構成は Step 6 のプレビューフォーマットに従うこと（基本情報、到達目標、ルーブリック、概要、注意点、授業計画、評価割合、MCC対応、授業の属性）。

## エクセルテンプレート構造

スキルが想定する6シート構成:

| シート名 | 主な内容 | 入力元 |
|----------|----------|--------|
| 科目情報・到達目標 | 基本情報、到達目標 | ユーザー事前入力 |
| ルーブリック | 評価観点と到達レベル | ユーザー事前入力 |
| 教育方法等・自由使用枠 | 概要、注意点、授業方法 | 一部ユーザー / 一部生成 |
| 授業計画 | 週ごとの内容と到達目標 | 生成 |
| 評価割合 | 評価方法と観点の配分 | 生成 |
| 授業の属性・履修上の区分 | AL、ICT等の属性 | 生成 |

### セル位置マッピング

読み取りスクリプト・書き込みスクリプトは以下のセル位置を使用する。テンプレートが変更された場合はスクリプト内の定数を修正する。

**科目情報・到達目標シート:**
- B1: 教科名, B4: 科目番号, D4: 科目区分
- B5: 授業形態, D5: 単位
- B6: 学科, D6: 学年, B7: 開講期, D7: 週時間数
- B8: 教科書/教材, B9: 担当教員
