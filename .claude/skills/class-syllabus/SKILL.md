# シラバス作成スキル

高専のシラバスをエクセルテンプレートとMCC（モデルコアカリキュラム）をもとに作成するスキル。ユーザーが入力した基本情報・到達目標・ルーブリックから、授業計画・評価割合・概要などを生成し、エクセルとMarkdownの両方に出力する。

## なぜこのスキルが必要か

高専のシラバス作成は、6つのシートにまたがるエクセルファイルへの記入が必要で、MCCとの対応付けも求められる。この作業は毎年繰り返されるが、到達目標やルーブリックから授業計画を導き出すプロセスは構造的で、支援の余地が大きい。

## 前提条件

- 1コマの授業時間は **90分** とする
- Python3 がインストールされていること
- 初回実行時に依存パッケージをインストールする:
  ```bash
  pip3 install -r ${CLAUDE_SKILL_DIR}/requirements.txt
  ```
- エクセルテンプレートは6シート構成（後述）

## 対話ルール

ユーザーへの質問・プレビュー・修正ループの作法は `specs/dialog-conventions.md` を正本とする。本スキルもこの規約に従う。

## 出力フォーマット契約

- **シラバス Markdown**: Step 5・Step 5.5・Step 8 で生成する Markdown は `specs/syllabus-markdown-schema.md` を**唯一の出力仕様の正本**とする。セクション構成・構造化テーブルの列仕様・レポート名/試験名の正規化は全て同ファイルに従う
- **ルーブリック**: `## ルーブリック` セクションの表構造は `specs/rubric-schema.md` § 1.1 に従う
- **出力ファイル配置**: ファイルパス・科目フォルダ命名は `specs/output-layout.md` に従う

後続スキル（`class-syllabus-parse` / `class-test` / `class-report` / `class-slides`）は本スキーマに従って読み取ることを前提としているため、スキーマから逸脱した出力は契約違反となる。

## 処理フロー

```
Step 0: 教員名・科目名・開講年度の取得
Step 0.5: フロー選択（従来フロー / MCCカテゴリ逆算フロー）
Step 1: Excelファイルの指定
  ├→ Excel有り → Step 2a: 読み取り → Step 2b: 既存情報の利用確認
  └→ Excel無し → Step 2c: 対話で基本情報収集
Step 2.5: 到達目標・ルーブリックの確認・生成
  └→ [逆算フロー] MCCカテゴリ選択 → トピック洗い出し → 到達目標生成
Step 3: 授業構成パラメータ（回数・授業構成案・評価スタイル・試験・MCC）
Step 3.5: 図作成課題の検討（作図適性の判定 → 評価課題/授業活動/両方/不要）
Step 4: MCC対応付け（オプション / 逆算フローでは自動適用）
Step 5: 生成（概要・授業の進め方・注意点・授業計画・評価割合・属性）
  └→ [逆算フロー] 5d. MCCトピック配分ロジックで週次計画を逆算
Step 5.5: 試験範囲の確認（小テスト・定期試験がある場合）
Step 6: プレビュー・修正ループ
Step 7: Excel書き込み（Excel有りの場合）
Step 8: Markdown出力
Step 9: 完了レポート
```

### Step 0: 教員名・科目名・開講年度の取得

最初に必ず教員名と科目名を確認する。これらはシラバスの基本であり、以降の全ステップで使用する。必ず1つずつ段階的に質問する（まとめて聞かない）。

まずプレーンテキストで教員名を質問する:
```
担当教員のお名前を教えてください:
```

回答を受けてから、科目名を質問する:
```
科目名を教えてください:
```

回答を受けてから、開講年度を質問する:
```
開講年度を教えてください（例: 2026）:
```

年度は教科書/教材のデフォルトURL生成（`https://moodle[YYYY].center.yuge.ac.jp`）に使用する。

### Step 0.5: フロー選択

`AskUserQuestion` で確認する:
- options: 「到達目標をもとに授業計画を組み立てる（従来フロー）」「MCCカテゴリを起点に授業内容を逆算する（逆算フロー）」

選択結果を **フロー種別** として記録し、以降の Step 2.5・Step 3・Step 4・Step 5d での分岐に使用する。どちらを選んでも Step 1 へ進む。

### Step 1: Excelファイルの指定

プレーンテキストで質問する:
```
シラバスのエクセルテンプレートファイルのパスを教えてください
（ない場合は「なし」と入力してください）:
```

- パスが指定された場合 → Step 2a へ
- 「なし」の場合 → Step 2c へ

### Step 2a: Excel読み取り（Excel有りの場合）

同梱の読み取りスクリプトでデータを抽出する:

```bash
python3 ${CLAUDE_SKILL_DIR}/scripts/read_syllabus.py "<エクセルファイルパス>"
```

出力はJSON形式。スクリプトがエラーを返した場合（ファイル不在、シート不足など）はユーザーに伝えて対応を求める。

抽出した情報を整理して表示する:

```
=== シラバス基本情報 ===
教科名: [科目名]
科目番号: [番号]
科目区分: [区分]
授業形態: [形態]
単位: [単位]
開設学科: [学科]
対象学年: [学年]
開講期: [期]
週時間数: [時間]
教科書/教材: [教材]
担当教員: [教員名]

=== 到達目標 ===
（読み取った到達目標を表示）

=== ルーブリック ===
| 評価観点 | 理想的 | 標準的 | 未到達 |
|----------|--------|--------|--------|
| ... | ... | ... | ... |

=== 評価割合テンプレート ===
評価方法: [B1], [C1], [D1], [E1], [F1], [G1], [H1]
観点: [A2], [A3], [A4], [A5]
```

**評価割合テンプレートの確認**: 読み取った評価方法ヘッダーと観点名は、テンプレートによって異なる場合がある（例: 「定期試験」→「試験」、「汎用的技能」→「思考・推論・創造への適応力」など）。ここで読み取った値は Step 3 の評価方法選択と Step 5e の評価割合生成で使用する。

**教員名・科目名の整合チェック**: Excel から読み取った値が Step 0 で入力された値と異なる場合、`AskUserQuestion` でどちらを使うか確認する:
- options: 「Step 0で入力した値を使う」「Excelの値を使う」

**教科書/教材の確認**:
読み取った教科書/教材が空でない場合、`AskUserQuestion` で確認する:
- options: 「この内容のまま使う」「変更する」
- 「変更する」を選んだ場合、プレーンテキストで新しい値を質問する

読み取った教科書/教材が空の場合、プレーンテキストで以下を提案する:
```
教科書/教材が設定されていません。デフォルトとして https://moodle[YYYY].center.yuge.ac.jp を設定しますか？
（変更する場合は入力してください）:
```

整合に問題がなければ、「この内容で合っていますか？修正があれば教えてください。」とプレーンテキストで確認する。

### Step 2b: 既存情報の利用確認（Excel有りの場合）

Excel に概要や授業計画が既に記入済みの場合、`AskUserQuestion` で確認する:

```
テンプレートに以下の既存情報が見つかりました:
- 概要: （記入済み / 空）
- 授業計画: （記入済み / 空）
- 評価割合: （記入済み / 空）
- 授業の属性: （記入済み / 空）
```

- options: 「既存情報をベースに修正・補完する」「既存情報を無視して新規作成する」

全て空の場合はこのステップをスキップする。

**「既存情報を無視して新規作成する」が選択された場合**: Step 2.5a では「到達目標が存在しない場合」の分岐に進む（Excelから読み取った到達目標・ルーブリックは使用しない）。

確認後 → Step 2.5 へ

### Step 2c: 対話で基本情報収集（Excel無しの場合）

以下をプレーンテキストで収集する:

```
以下の基本情報を教えてください:
- 科目番号:
- 科目区分（例: 専門/必修）:
- 授業形態（例: 授業、演習）:
- 単位の種別と単位数（例: 履修単位: 1）:
- 開設学科:
- 対象学年:
- 開講期（例: 前期、後期、通年）:
- 週時間数:
- 教科書/教材（空欄の場合は https://moodle[YYYY].center.yuge.ac.jp をデフォルトとして提案する）:
```

到達目標とルーブリックは Step 2.5 で収集・生成する。

収集完了後 → Step 2.5 へ

### Step 2.5: 到達目標・ルーブリックの確認・生成

#### Step 2.5a: 到達目標の確認・生成

**[逆算フロー]の場合**: 到達目標の有無にかかわらず、`references/mcc-reverse-flow-guide.md` の Step A〜C に従い、MCCカテゴリ選択 → トピック洗い出し → 到達目標生成の順に進める。到達目標確定後、Step 2.5b へ進む。

**[従来フロー]の場合**: 以下の通り、到達目標の状態に応じて分岐する。

**到達目標が存在する場合**（Excel読み取り済み、または既存情報をベースに選択した場合）:

現在の到達目標を表示した上で、`AskUserQuestion` で確認する:
- options: 「この到達目標のまま進める」「改善・再生成したい」「一から書き直したい」

「改善・再生成したい」「一から書き直したい」のいずれかが選ばれた場合、散文形式（段落文章）で生成する。ユーザーが明示的に箇条書き形式を希望した場合のみ箇条書きで生成する。

- 「改善・再生成したい」→ `references/objective-generation-guide.md` の改善モードに従い改善案を提示する
- 「一から書き直したい」→ `references/objective-generation-guide.md` の生成手順に従い到達目標を生成する。またはプレーンテキストでユーザーが直接入力する

**到達目標が存在しない場合**（Excelに空、または Excel無し）:

`AskUserQuestion` で方式を確認する:
- options: 「自分で入力する」「科目情報から自動生成してほしい」「MCCカテゴリを指定して生成してほしい」

「自分で入力する」→ プレーンテキストで収集:
```
この科目の到達目標を教えてください:
（複数ある場合は番号付きで入力してください）
```

「科目情報から自動生成してほしい」「MCCカテゴリを指定して生成してほしい」のいずれかが選ばれた場合、散文形式（段落文章）で到達目標を生成する。ユーザーが明示的に箇条書き形式を希望した場合のみ箇条書きで生成する。`references/objective-generation-guide.md` の生成手順に従い提案する。

生成結果の確認は `AskUserQuestion` で行う:
- options: 「この到達目標で確定する」「一部を修正したい」「再生成してほしい」

#### Step 2.5b: ルーブリックの確認・生成

**ルーブリックが存在する場合**（Excel読み取り済み）:

ルーブリックを表示し、プレーンテキストで「この内容で合っていますか？修正があれば教えてください。」と確認する。

**ルーブリックが存在しない場合**:

`AskUserQuestion` で方式を確認する:
- options: 「自分で入力する」「到達目標から自動生成してほしい」

「自分で入力する」の場合:
```
各評価観点について、理想的・標準的・未到達の基準を教えてください:

評価観点1:
- 理想的な到達レベル:
- 標準的な到達レベル:
- 未到達レベル:
```

「自動生成」の場合: Step 2.5a で確定した到達目標の内容からルーブリックを生成し、プレーンテキストで確認を取る。

確認完了後 → Step 3 へ

### Step 3: 授業構成パラメータの確認

以下を順に確認する（Excel有無にかかわらず必ず全項目を質問する）:

**授業回数**: 逆算フローで Step B にて既に確認済みの場合はスキップする。未確認の場合はプレーンテキストで質問する。
```
授業回数は何回ですか？（例: 15, 16, 30, 32）:
```

**授業構成案**: `AskUserQuestion` で授業スタイルを質問する。
- options: 「講義中心」「演習中心」「講義＋演習」「反転授業」

複合型（「講義＋演習」など講義と演習の両方を含むスタイル）が選ばれた場合、プレーンテキストで時間配分を質問する:
```
1コマ90分のうち、講義と演習の配分はどのくらいを想定していますか？
（例: 講義60分＋演習30分、講義45分＋演習45分）:
```

授業構成案と時間配分は、Step 5c の授業計画生成に反映する。各週の授業内容を記述する際に、構成に沿った流れ（例: 前半で概念を講義→後半で演習問題に取り組む）を意識して計画を組み立てる。

**使用する評価方法**: `AskUserQuestion`（multiSelect: true）で質問する。
- **Excel有りの場合**: Step 2a で読み取った評価割合テンプレートの評価方法ヘッダー（B1〜H1の値）を選択肢として表示する。テンプレートに存在する評価方法のみが選択可能。
- **Excel無しの場合**: まず `AskUserQuestion` で評価方法・観点のテンプレートパターンをユーザーに選んでもらう（パターンの詳細は `references/json-format.md` の「テンプレートパターン」セクションを参照）。選択されたパターンの評価方法名を選択肢として表示する。デフォルト（パターンA）の場合:
  - options: 「定期試験」「小テスト」「レポート・課題」「口頭発表」「成果物・実技」「ポートフォリオ」「その他（出席点・授業態度等）」

選択された評価方法ごとのフォローアップ質問と重み付け方針の詳細は `references/evaluation-guide.md` を参照。フォローアップ質問はテンプレートのヘッダー名に関わらず、評価方法の性質（試験系・レポート系・発表系など）に応じて適切なものを適用する。

**MCC対応**:
- **[従来フロー]**: `AskUserQuestion` で質問する:
  - options: 「MCCとの対応付けを行う」「MCCとの対応付けは不要」
- **[逆算フロー]**: Step 2.5 で確定したMCCカテゴリを自動適用する。「Step 2.5 で選択した [カテゴリ名] をMCC対応に使用します」と通知し、`AskUserQuestion` で確認する:
  - options: 「そのまま使う」「カテゴリを変更したい」

### Step 3.5: 図作成課題の検討

`class-slides` スキルは運用の安定化のためスライド内で図や写真を原則使わず、表現が難しい複雑図のみ mermaid コードのベタ貼りで扱う。この制約を補うため、フロー・構造・関係などの説明に必要な図を**学生自身が作図する**ことを、評価課題または授業中の作図演習として組み込むかをここで検討する。

手順の詳細は `references/figure-task-guide.md` に委譲する。本ステップでは以下の流れで進める:

1. **作図適性の意味的判定**（根拠提示つき）: 到達目標・授業構成案・（生成済みなら）授業計画から、学生が作図することで理解が深まりそうな箇所を抽出する。判定はキーワード一致ではなく意味から行い、候補は必ず根拠（どの到達目標・どの週か）とともに提示する
2. **取り扱い方針の確認**: `AskUserQuestion` で「評価課題として組み込む」「授業中の作図演習として組み込む（評価対象外）」「両方を組み合わせる」「図作成課題は設けない」から選択させる
3. **対象回・件数の確定**: 評価課題の件数は 2〜3 件を目安とし、3 件を超える場合は warning を出して再考を促す
4. **作図ツール・提出形式の確認**（評価課題の場合）: `AskUserQuestion` で「mermaid」「PowerPoint」「指定なし（学生任意）」「その他（自由記述）」から選択させる。**提出可能な拡張子は `.md` / `.txt` / `.pdf` に限定**する（ツール選択に関わらず適用）
5. **評価割合への反映**（評価課題の場合）: `AskUserQuestion` で「既存の『レポート・課題』または『成果物・実技』に含める」「独立項目として追加する（割合を再配分）」から選択させる。独立項目追加時は合計 100% の検算を必ず行う
6. **授業計画への反映**: 授業活動は該当週の授業内容末尾に `作図演習: [概念名]` を構造化追記する。評価課題は通常のレポート・成果物と同じ扱いで提出週に「[課題名]の提出」を記載する
7. **ルーブリックへの反映**（評価課題の場合）: Step 2.5b で生成済みの `## ルーブリック` セクションに観点を追加する。観点は Step 3.5-a の根拠として引用した到達目標から導出し、追加は 1〜2 行を目安とする

「図作成課題は設けない」が選ばれた場合、3〜7 はスキップして Step 4 に進む。各ステップの詳細フォーマット・文言・フォールバック処理は `references/figure-task-guide.md` を参照する。

### Step 4: MCC対応付け

- **[従来フロー]**: Step 3 で MCC 対応を選択した場合のみ実施。詳細手順は `references/mcc-guide.md` を参照。
- **[逆算フロー]**: `references/mcc-reverse-flow-guide.md` の Step E に従い、選択済みカテゴリを自動適用して確認する。

### Step 5: 授業計画・出力項目の生成

到達目標・ルーブリック・MCCの分析結果をもとに、概要・授業の進め方・注意点・授業計画・評価割合・授業の属性を生成する。各項目の生成ルールは `references/course-design-guide.md` を参照。

**[逆算フロー]の授業計画生成（5d）**: `references/mcc-reverse-flow-guide.md` の Step D（週次配分ロジック）に従い、Step 2.5 で確定したカテゴリ別トピックリストを週次に配分して授業計画を生成する。概要・授業の進め方・注意点・評価割合・授業の属性は従来フローと同じルールで生成する。

概要の出力形式の確認・生成ルールは `references/course-design-guide.md` の 5a を参照。授業の進め方・注意点は箇条書き（階層付き）をデフォルトとする（詳細は `references/teaching-method-guide.md` を参照）。

**評価割合の生成（テンプレート準拠）**: Excel有りの場合、評価割合の生成時には Step 2a で読み取ったテンプレートの**評価方法ヘッダー名**と**観点名**をそのまま使用する。JSONデータのキー名もテンプレートのヘッダー・観点名と一致させること（`references/json-format.md` 参照）。Excel無しの場合は標準の評価方法名・観点名を使用する。

評価割合の生成後は、全観点・全評価方法の配分値の合計が100になっていることを必ず検算すること。100でない場合は調整してからプレビューに進む。

### Step 5.5: 試験範囲の確認

Step 3 で小テストまたは定期試験が選択されている場合に実施する。いずれも選択されていない場合はスキップする。

Step 5 で生成した授業計画をもとに、各試験の対象範囲を自動割り当てし、ユーザーに確認を取る。確定した範囲は授業計画の授業内容テキストと「授業の進め方」テキストに反映する。

**`（要確認）` の扱い**: 授業計画から範囲を自動割り当てできない場合（該当週が未確定など）は、試験範囲に `（要確認）` を記載する。Step 6 のプレビューまでに確定させるのが原則だが、どうしても確定しない場合は `（要確認）` のまま出力することを許容する。後続スキル（`class-test` 等）が二重ガードで再確認するため、シラバス全体の作成を止めずに進められる（詳細は `specs/syllabus-markdown-schema.md` § 4.4 参照）。

**試験名の正規化（`定期試験` の判定）**: Markdown 出力時に試験スケジュール全体を見て、`定期試験` という名称が含まれる場合の正規化ルールを適用する。同一スケジュール内に `中間試験` が存在しない場合のみ `定期試験` を `期末試験` に置換する。中間試験と併存する場合は `定期試験` のまま残す（2回実施の後半として扱う）。詳細は `specs/syllabus-markdown-schema.md` § 5.2 参照。

詳細な自動割り当てロジック・確認フロー・反映先は `references/exam-scope-guide.md` を参照。「授業の進め方」への反映フォーマットは `references/teaching-method-guide.md` を参照。

### Step 6: プレビューと修正

生成した全内容を `references/preview-format.md` のフォーマットに従ってMarkdown形式でプレビュー表示する。

**書き込み対象外項目の説明**: プレビュー表示の前に、Excel自動書き込みの対象外となる項目をユーザーに説明する:

```
以下の項目はExcelへの自動書き込み対象外です（Excel側に既に記入されているか、手動での確認・記入が必要です）:
- 科目番号、科目区分、授業形態、単位、開設学科、対象学年、開講期、週時間数、担当教員（科目情報・到達目標シート）
- 学科の到達目標項目との関係（科目情報・到達目標シート A16）
これらを確認・修正する場合は、Excelファイルを直接編集してください。
```

`AskUserQuestion` で確認する:
- options: 「問題ない、確定する」「授業計画を修正したい」「評価割合を修正したい」「概要・授業の進め方・注意点を修正したい」

修正が選択された場合は、プレーンテキストで具体的な修正内容を聞き、反映後に再度プレビューする。修正が完了するまで繰り返す。

評価割合の修正時は、修正後に合計が100になっていることを検算し、100でない場合はその旨をユーザーに伝えて再修正を求める。

### Step 7: ファイル出力（Excel有りの場合）

`AskUserQuestion` で出力形式を確認する:
- options: 「Markdown + エクセルの両方に出力する（推奨）」「エクセルのみに書き込む」「Markdownのみ出力する」

選択に応じた動作:
- 「Markdown + エクセルの両方」→ Excel書き込み（以下）を実行し、その後 Step 8 のMarkdown出力も実行する
- 「エクセルのみ」→ Excel書き込み（以下）を実行し、Step 8 はスキップする
- 「Markdownのみ」→ Excel書き込みをスキップし、Step 8 のMarkdown出力のみ実行する

「エクセルに書き込む」場合:

元ファイルは変更せず、科目専用フォルダにコピーしてからコピーに書き込む:
```bash
mkdir -p "output/[開講年度]_[教科名]"
cp "<元ファイル>" "output/[開講年度]_[教科名]/<元ファイル名>"
python3 ${CLAUDE_SKILL_DIR}/scripts/write_syllabus.py "output/[開講年度]_[教科名]/<元ファイル名>" '<確定データJSON>'
```

例: 2026年度のアルゴリズムの場合 → `output/2026_アルゴリズム/`

スクリプトの仕様:
- 既存のセル書式（フォント、罫線、セル結合など）を保持する
- 値の書き込みのみ行う
- 書き込み対象: 科目情報・到達目標シート（到達目標・教科書/教材）、ルーブリックシート、教育方法等シート（概要・授業の進め方・注意点）、授業計画シート、評価割合シート、授業属性シート

確定データJSONのフォーマットは `references/json-format.md` を参照。

Excel無しの場合はこのステップをスキップする。

### Step 8: Markdown出力

Markdownファイルは科目専用フォルダに保存する:
```bash
mkdir -p "output/[開講年度]_[教科名]"
```

ファイル名:
- Excel有りの場合: `output/[開講年度]_[教科名]/[エクセルファイル名（拡張子なし）].md`（例: 2026年度・アルゴリズム・`26開講情報工学科3アルゴリズム.xlsx` → `output/2026_アルゴリズム/26開講情報工学科3アルゴリズム.md`）
- Excel無しの場合: `output/[開講年度]_[教科名]/[教科名].md`

Step 6 のプレビュー内容をそのままMarkdownファイルとして保存する。出力するセクション構成は Step 6 のプレビューフォーマットに従うこと（基本情報、到達目標、ルーブリック、概要、授業の進め方、注意点、授業計画、評価割合、評価の計算方法・計算例、MCC対応、授業の属性）。

```

生成した項目:
```
