# 授業の進め方 生成ガイド

Step 5b で「授業の進め方」テキストを生成する際の構造化フォーマットと記述ルール。

## 出力形式

**箇条書き（階層付き）をデフォルト**とする。

### 箇条書きルール

- 中黒 `・` で項目を列挙する
- 階層は最大3段階まで。インデントに全角スペースを使用する:
  - 第1階層: `・項目`
  - 第2階層: `　・子項目`（全角スペース1つ + 中黒）
  - 第3階層: `　　・孫項目`（全角スペース2つ + 中黒）
- 1つの箇条書き項目は1文〜2文程度に収める
- 子項目が必要な場面: 前半/後半の構成分け、複数の提出物の列挙、試験の各回の範囲など
- 孫項目が必要な場面: 子項目の中でさらに詳細を分ける場合（例: 各レポートの個別説明、各試験回の範囲と方式）

## 構成要素

「授業の進め方」は以下の要素を順に記載する。**選択されていない評価方法に対応する要素は省略する。**

| 要素 | 条件 | 内容 |
|------|------|------|
| ①授業スタイル | 必須 | 講義・演習の配分と進め方 |
| ②教材 | 必須 | 教材の配布・利用方法 |
| ③小テスト | 小テストを選択した場合 | 実施回数・方式・範囲一覧 |
| ④定期試験 | 定期試験を選択した場合 | 実施時期・範囲 |
| ⑤成果物・課題 | 成果物・実技またはレポート・課題を選択した場合 | 主要課題の概要と期間 |

## 各要素の記述ルール

### ①授業スタイル

Step 3 で収集した授業構成案・時間配分を反映する。授業スタイルの概要を第1階層に、前半/後半などの内訳を第2階層に記載する。SKILL.md Step 3 の選択値に応じて以下のテンプレートを使う。選択肢外が指定された場合も同じ枠組み（第1階層で全体方針、第2階層で内訳）で記述を組み立てる。

| 授業構成案 | 第1階層 | 第2階層（例） |
|------------|---------|---------------|
| 講義中心 | 「講義を中心に進める。」 | 必要に応じて前半/後半の構成 |
| 演習中心 | 「演習を中心に進める。」 | 必要に応じて前半/後半の構成 |
| 講義＋演習 | 「毎回の授業は前半[X]分を講義、後半[Y]分を演習に充てる講義＋演習形式で進める。」 | 前半/後半の内容の概要 |
| 反転授業 | 「事前学習を前提とした反転授業形式で進める。」 | 事前学習と授業内活動の内訳 |

### ②教材

教科書/教材の値に応じて記述する。

- Moodle URLの場合: 「教材はMoodle（[URL]）で配布する。」
- 書籍の場合: 「教科書として[書籍名]を使用する。」
- 複数ある場合は第2階層で列挙する。

### ③小テスト

**試験範囲が確定している場合**（Step 5.5 完了後）に記載する。第1階層に概要、第2階層に各回の詳細を記載する。

### ④定期試験

**試験範囲が確定している場合**（Step 5.5 完了後）に記載する。第1階層に概要、中間・期末が両方ある場合は第2階層で分ける。

### ⑤成果物・課題

主要な成果物や課題がある場合に記載する。第1階層に概要、第2階層で個別の課題を列挙する。

## 構造化サブセクション

箇条書きの本文に加え、以下の構造化テーブルを「授業の進め方」セクションの末尾（「注意点」セクションの手前）に必ず挿入する。これらのテーブルは `/class-syllabus-parse`・`/class-test`・`/class-report` スキルがスケジュール情報を機械的に抽出する際の入口となる。

### `### レポート・課題スケジュール`

レポート・成果物が評価に含まれる場合に挿入する。

- テーブルの列は固定で `レポート名`、`提出週`、`テーマ` の3列とする
- `レポート名` はアラビア数字で表記する（`①②③` や `第1レポート` のような表記は使わない）
  - 複数の通常レポート: `レポート1`、`レポート2`、…
  - 最終レポート・中間レポート・期末レポート等の特殊名はそのまま使用してよい
- `提出週` は `第[N]週` の形式とする
- `テーマ` は学生が何に取り組むかが分かる具体的な名前を書く
- 箇条書きとテーブルの内容は整合させる（同じレポート数・同じ提出週・同じテーマ）

### `### 試験スケジュール`

小テストまたは定期試験が評価に含まれる場合に挿入する。

- テーブルの列は固定で `試験名`、`実施週`、`範囲` の3列とする
- `試験名` はアラビア数字で表記する（`①②③` や `第1回` のような表記は使わない）
  - 複数の小テスト: `小テスト1`、`小テスト2`、…
  - 中間試験・期末試験等の特殊名はそのまま使用してよい
- `実施週` は `第[N]週` の形式とする
- `範囲` は `第[X]〜[Y]週` の形式または `全範囲（第[X]〜[Y]週）` の形式とする
  - 範囲が確定していない時点では `（要確認）` と記載してよい（Step 5.5 で確定させる）
- 箇条書きとテーブルの内容は整合させる（同じ試験数・同じ実施週・同じ範囲）

## 生成タイミング

「授業の進め方」テキストは2段階で生成する:

1. **Step 5（初回生成）**: ①授業スタイル、②教材を生成する。③小テスト・④定期試験は試験範囲が未確定のため、仮テキスト（回数・頻度のみ）で生成する。⑤成果物・課題は授業計画（Step 5d）の生成後に、対応する週を参照して記述する
2. **Step 5.5（範囲反映）**: 試験範囲の確定後、③小テスト・④定期試験の記述を範囲情報を含む確定テキストに更新する

## 出力例

### 例1: 講義＋演習・レポート・チーム開発あり（試験なし）

```
・毎回の授業は前半30分を講義、後半60分を演習に充てる講義＋演習形式で進める。
　・前半（1〜7週）: Markdownやgit/GitHubの基本を講義と演習で習得する。
　・後半（8〜14週）: チームでのシステム開発プロジェクトを実践する。
・教材はMoodle（https://moodle2026.center.yuge.ac.jp）で配布する。
・提出物は以下の通りである。
　・レポートは単元ごとに3回提出する。
　・チーム開発期間中（9〜14週）はKPT形式の自己評価を毎週提出する。
　・全ての提出物について生成AI活用のプロンプト記録を求める。
・学修単位のため、1単位あたり30時間の自学自習が必要である。自学自習の内容は授業中に指示する。

### レポート・課題スケジュール
| レポート名 | 提出週 | テーマ |
|-----------|--------|-------|
| レポート1 | 第2週 | 好きなチェーン店の解説と分析 |
| レポート2 | 第3週 | 任意の料理の手順書 |
| レポート3 | 第15週 | 改善提案レポート |
```

### 例2: 講義＋演習・小テスト・期末試験・成果物あり

```
・毎回の授業は前半60分を講義、後半30分を演習に充てる。
・教材はMoodle（https://moodle2026.center.yuge.ac.jp）で配布する。
・小テストは単元ごとに3回実施する。
　・第5週: 第1〜4週範囲
　・第8週: 第5〜7週範囲
　・第10週: 第8〜9週範囲
・期末試験は第15週に実施し、全範囲（第1〜14週）を対象とする。
・第11〜14週でvsCPU形式のリバーシプログラムの開発に取り組む。

### 試験スケジュール
| 試験名 | 実施週 | 範囲 |
|--------|--------|------|
| 小テスト1 | 第5週 | 第1〜4週 |
| 小テスト2 | 第8週 | 第5〜7週 |
| 小テスト3 | 第10週 | 第8〜9週 |
| 期末試験 | 第15週 | 全範囲（第1〜14週） |
```

### 例3: 講義中心・中間試験・期末試験あり

```
・講義を中心に進める。
・教科書として「アルゴリズムとデータ構造」を使用する。補助教材はMoodle（https://moodle2026.center.yuge.ac.jp）で配布する。
・中間試験は第8週に実施し、第1〜7週を対象とする。
・期末試験は第16週に実施し、全範囲（第1〜15週）を対象とする。

### 試験スケジュール
| 試験名 | 実施週 | 範囲 |
|--------|--------|------|
| 中間試験 | 第8週 | 第1〜7週 |
| 期末試験 | 第16週 | 全範囲（第1〜15週） |
```
