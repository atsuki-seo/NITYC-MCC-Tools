# 授業用 Marp スライド生成スキル

シラバス Markdown の授業計画・到達目標・ルーブリック・試験/レポートスケジュールをもとに、`theme: yuge` の Marp スライドを回ごとに生成するスキル。

## なぜこのスキルが必要か

高専の授業ではシーズン開始前に全授業回分のスライドを準備する必要がある。シラバスには回ごとの内容・到達目標・MCC 対応・提出物スケジュールが定義済みなので、それらを入力に各回のスライド骨子と本体を自動生成することで教員の資料作成負荷を削減できる。1回分の単発生成とシーズン全回一括生成の両方に対応する。

## 前提条件

- シラバス Markdown ファイルが `output/` 配下に存在すること（`/class-syllabus` スキルで事前に作成）
- Marp レイアウトテーマ `yuge` が別途インストール済みであること（本スキルは CSS を生成しない）
- 出力はすべて Marp 形式 Markdown

## デフォルト設定

| 項目 | デフォルト |
|------|-----------|
| Marp テーマ | `yuge`（固定） |
| ファイル単位 | 1回 = 1 Marp ファイル |
| 情報量目安 | 30分=10枚 / 60分=20枚 / 90分=30枚（`references/slide-structure-guide.md` 参照） |
| 図解方式 | mermaid のみ（写真・イラストは挿入しない） |
| 提出物展開 | 該当回は tests/reports から詳細展開（埋め込み型） |

ユーザーが明示的に変更しない限り、上記を適用する。

## 対話ルール

ユーザーへの質問は場面に応じて適切な方式を使い分ける:

- **自由入力が自然な場面**（ファイルパス、修正指示の内容など）: プレーンテキストで質問する
- **選択肢が明確な場面**（対象回の範囲、授業時間、上書き確認など）: `AskUserQuestion` ツールを使う

不明点は推測せずユーザーに確認する。

## 処理フロー

```
Step 0: シラバス読み込み（class-syllabus-parse 委譲）
Step 1: 対象範囲の確認（全回 / 特定回）
Step 2: 授業時間と情報量の確認
Step 3: 全回の骨子生成
Step 4: 骨子プレビューと修正ループ
Step 5: 回ごとの詳細化ループ（生成→プレビュー→修正→出力→次へ）
Step 6: 完了レポート
```

### Step 0: シラバス情報の読み込み

シラバス解析は `class-syllabus-parse` スキルに委譲する。会話コンテキストに既にシラバス情報が読み込まれているかを確認し、以下のいずれかで処理する:

- **既に `class-syllabus-parse` で読み込み済みの場合**: 抽出結果を再利用して Step 1 へ進む
- **読み込まれていない場合**: ユーザーに `class-syllabus-parse` スキルの事前実行を案内する:
  ```
  授業スライドの作成にはシラバス情報の解析が必要です。
  先に `/class-syllabus-parse` スキルを実行してシラバスを読み込んでください。
  ```

シラバスMarkdownがそもそも存在しない場合は `/class-syllabus` スキルでの事前作成を案内する。

`class-syllabus-parse` 実行後、コンテキストには以下の情報が残っている前提で Step 1 以降を進める（コンテキスト出力契約の正本は `specs/syllabus-markdown-schema.md` § 9.1）:

- 基本情報: 科目名・開講年度・担当教員・対象学年・開講期・単位・教科書/教材・開設学科・評価割合（原文名＋カテゴリキー）
- 授業計画: 各週の授業内容・到達目標対応（全週分をコンテキスト保持）
- 到達目標・ルーブリック
- レポート・試験スケジュール（正規化済みの名前・週・範囲）
- 概要: `## 概要` セクション本文（原文保持）
- MCC対応: `## MCC対応` テーブル原文
- 授業の属性: `## 授業の属性` テーブル原文

授業時間は本スキル Step 2 でユーザーに確認するため parse では保持しない。

### Step 1: 対象範囲の確認

`AskUserQuestion` で範囲を確認する:

- options: 「シーズン全回一括」「特定回のみ」

「特定回のみ」が選択された場合、プレーンテキストで回番号を聞く（複数可: 例「1,3,5」「7-9」）。

指定回番号がシラバス授業計画にない場合は有効な回番号リストを提示してやり直す。

**重要**: 特定回モードでも、シラバス全回の授業計画情報は Step 3 で読み込む。前後参照・用語統一のため。

### Step 2: 授業時間と情報量の確認

シラバスに授業時間（30分/60分/90分）の記載があれば自動採用する。ない場合は `AskUserQuestion` で確認する:

- options: 「30分」「60分」「90分」

授業時間と枚数目安の対応は `references/slide-structure-guide.md` の §1 に従う。

### Step 3: 全回の骨子生成

対象範囲の各回について以下を作成する:

- 回番号 / 回タイトル
- 授業時間 / 枚数目安
- キーコンセプト（3〜5個）
- 節構成（章区切りスライドに対応する節の名前、1〜3個）
- 含める要素: 表紙 / 目次 / 章区切り / 導入 / 概念解説 / 比較表 / 例題 / 提出物 / まとめ（各要素の枚数を明示）
- 到達目標参照（ルーブリック項目番号）
- MCC 対応（V-D-*）
- 提出物該当回の場合: 参照するファイルパス（`output/[YYYY]_[教科]/reports/[レポート名]/` や `tests/[テスト名]/`）

**提出物該当回の判定**: シラバスのレポート・試験スケジュールから「提出週」と一致する回を自動判定する。該当回について以下のファイル存在を確認する:

- `output/[YYYY]_[教科名]/reports/[レポート名]/[レポート名]_課題.md`
- `output/[YYYY]_[教科名]/reports/[レポート名]/[レポート名]_ルーブリック.md`
- `output/[YYYY]_[教科名]/tests/[テスト名]/[テスト名]_確認用.md`

存在しないファイルがある場合は骨子に「簡略版」フラグを付与し、Step 4 の骨子表示で警告を出す。

### Step 4: 骨子プレビューと修正ループ

全回の骨子一覧を Markdown テーブルで提示する:

```
=== 骨子案 ===
| 回 | タイトル | 時間 | 枚数 | 節構成 | キーコンセプト | 到達目標 | MCC | 提出物 |
|----|---------|------|------|--------|---------------|---------|-----|-------|
| 1 | ... | 60分 | 20 | ... | ... | ... | ... | — |
| 2 | ... | 60分 | 18 | ... | ... | ... | ... | レポート1 |
| ... |
```

簡略版フラグがある行は「（簡略版: [ファイル名] 未発見）」と注記する。

`AskUserQuestion` で確認する:
- options: 「この骨子で進む」「回を指定して修正したい」「全体を再生成したい」

「回を指定して修正したい」はプレーンテキストで修正指示を聞き（例「第3回にlex実習を足して、例題を2スライド増やして」）、該当回の骨子を更新して再提示する。確定するまで繰り返す。

### Step 5: 回ごとの詳細化ループ

対象回数分、以下を繰り返す（骨子順）。

#### Step 5.1: 該当回のスライド本体を生成

骨子に従ってスライドごとの Markdown を書く。生成ルール:

- `references/output-format.md` のファイル構造テンプレートに従う
- `references/yuge-theme-layout.md` のレイアウト規約に従う（`flex-layout` / `columns` / Tips 囲み / mermaid）
- `references/slide-structure-guide.md` の情報量ルール（箇条書き7〜10行、1行35文字以内、枚数目安±2枚）を遵守
- 提出物該当回: `reports/*/*_課題.md` / `*_ルーブリック.md` / `tests/*/*_確認用.md` から課題名・期限・概要・要件・評価観点を読み取って展開する
- 簡略版フラグがある場合: シラバスのスケジュール記載のみで作成
- 導入スライドに到達目標番号、まとめスライドに MCC 項目を明示

#### Step 5.2: プレビュー提示

以下を表示する:

- 当該回の全スライドのタイトル一覧
- 代表スライド（表紙・概念解説の主要1枚・まとめ）の本文
- 生成枚数 vs 目安

#### Step 5.3: 修正指示ループ

`AskUserQuestion` で確認する:
- options: 「この内容で出力」「内容を修正したい」「再生成したい」

「内容を修正したい」はプレーンテキストで修正指示を聞き（例「第2スライドを2カラムに」「例題を増やして」）、差分をハイライトで提示しつつ更新する。確定するまで繰り返す。

#### Step 5.4: ファイル出力

出力先を作成する:

```bash
mkdir -p "output/[開講年度]_[教科名]/slides"
```

ファイル名: `classNN.md`（NN は2桁ゼロ埋めの回番号）。例: 2026年度コンパイラの第1回 → `output/2026_コンパイラ/slides/class01.md`

既に同名ファイルが存在する場合の挙動は `specs/file-output-policy.md` に従う。本スキルは § 1.2 のファイル単位で判定し、複数回の連続生成のため § 2.2 で許可される「スキップ」を選択肢に追加する。最終的な選択肢は「上書き」「別名で保存（`classNN_v2.md` 等、§ 3 のサフィックス規則）」「スキップ（この回を出力せず次の回へ進む）」「キャンセル（連続生成自体を中断）」。

#### Step 5.5: 次回への進行確認

`AskUserQuestion` で確認する:
- options: 「次の回（第N+1回）に進む」「ここで中断する」

「中断する」が選択された場合は Step 6 へ。対象回がすべて完了したら自動的に Step 6 へ。

### Step 6: 完了レポート

```
=== 授業スライド生成完了 ===
科目名: [科目名]
担当教員: [教員名]
開講年度: [YYYY]

生成したスライド:
| 回 | タイトル | ファイル | 枚数 |
|----|---------|---------|------|
| 1 | ... | output/YYYY_[教科]/slides/class01.md | 20 |
| ... |

未処理の回（ある場合）:
- 第N回 / 第N+1回 / ...

プレビュー方法:
  marp --watch output/YYYY_[教科]/slides/classNN.md

PDF/PPTXエクスポート:
  marp output/YYYY_[教科]/slides/classNN.md --pdf
  marp output/YYYY_[教科]/slides/classNN.md --pptx
```
