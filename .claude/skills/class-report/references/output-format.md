# 出力ファイル フォーマット仕様

Step 6 のプレビュー表示および Step 7 のファイル出力で使用するフォーマット。

## 出力ディレクトリ構造

```
output/[年度]_[教科名]/reports/[レポート名]/
├── [レポート名]_課題.md          ← 学生向け出題文
├── [レポート名]_ルーブリック.md    ← 採点用ルーブリック
└── submissions/                  ← 提出物配置フォルダ（空で生成）
```

`submissions/` は教員がMoodleからダウンロードした学生の提出物を配置するためのフォルダ。`/class-report-check` スキルで採点する際にこのフォルダを指定する。

## 課題文 Markdown フォーマット

ファイル名: `[レポート名]_課題.md`

```markdown
# [レポート名]: [科目名]

## 基本情報
| 項目 | 内容 |
|------|------|
| 科目名 | [科目名] |
| 担当教員 | [教員名] |
| 対象学年 | [学年] |
| レポート名 | [レポート名] |
| 提出期限 | 第[N]週 |
| 提出形態 | [個人 / チーム] |
| 提出形式 | [Markdown / PDF / プログラム等] |

## 課題の目的

[到達目標との対応を含む1〜2文の目的説明]

## 課題の内容

### テーマ
[テーマ名と概要説明]

### 要件

**必須要件:**
- [必須要件1]
- [必須要件2]
- ...

**推奨要件:**
- [推奨要件1]
- [推奨要件2]
- ...

### 取り組みの手順

1. [ステップ1の説明]
2. [ステップ2の説明]
3. [ステップ3の説明]
...

## 提出物の仕様

- ファイル形式: [形式]
- 命名規則: `学籍番号_氏名_レポート名.拡張子`（例: `22001_山田太郎_レポート1.md`）
- 提出先: [提出先]
- 分量の目安: [目安]

## 評価基準

| 評価観点 | 配点 | 概要 |
|----------|------|------|
| [観点1] | [N]点 | [1行の説明] |
| [観点2] | [N]点 | [1行の説明] |
| ... | ... | ... |
| **合計** | **100点** | — |

> 詳細な評価基準（ルーブリック）は別途配布します。
```

### チームレポートの追加セクション

チームレポートの場合、「提出物の仕様」の後に以下を追加する:

```markdown
## チーム提出に関する注意事項

- **提出ディレクトリ名**: `チーム名_メンバー氏名1_メンバー氏名2_メンバー氏名3`（例: `チームA_山田_佐藤_鈴木`）
- **提出ファイル名**: ディレクトリ内に `report.md` 等のファイルを配置する
- チームメンバー全員の氏名・学籍番号をレポート冒頭に記載すること
- 各メンバーの担当箇所を明記すること（例: 「第2章: 山田太郎」）
- [役割分担ありの場合] 役割分担表を含めること
- [共同作成の場合] 各メンバーの貢献内容を末尾に記載すること
```

## ルーブリック Markdown フォーマット

ファイル名: `[レポート名]_ルーブリック.md`

```markdown
# ルーブリック: [レポート名]（[科目名]）

## 基本情報
| 項目 | 内容 |
|------|------|
| 科目名 | [科目名] |
| レポート名 | [レポート名] |
| 配点 | 100点満点 |
| 提出形態 | [個人 / チーム] |

## 点数換算基準

| 段階 | 点数範囲（観点配点に対する比率） |
|------|-------------------------------|
| 理想的 | 80〜100%（標準点: 90%） |
| 標準的 | 60〜79%（標準点: 70%） |
| 未到達 | 0〜59%（標準点: 30%） |

## ルーブリック

| 評価観点 | 理想的 | 標準的 | 未到達 | 配点 |
|----------|--------|--------|--------|------|
| [観点1] | [理想的の記述] | [標準的の記述] | [未到達の記述] | [N]点 |
| [観点2] | [理想的の記述] | [標準的の記述] | [未到達の記述] | [N]点 |
| ... | ... | ... | ... | ... |
| **合計** | — | — | — | **100点** |

## 採点メモ欄

| 評価観点 | 評価 | 点数 | メモ |
|----------|------|------|------|
| [観点1] | □理想的 □標準的 □未到達 | /[N] | |
| [観点2] | □理想的 □標準的 □未到達 | /[N] | |
| ... | ... | ... | |
| **合計** | — | **/100** | |
```

---

## フォーマット規則

1. **セクション順序**: 基本情報 → 目的 → 内容 → 提出物 → 評価基準 の順で固定する
2. **表形式**: Markdownテーブルを使用し、列の揃えは左寄せとする
3. **合計行**: 配点テーブルには必ず合計行を含める
4. **エンコーディング**: UTF-8、BOMなし
