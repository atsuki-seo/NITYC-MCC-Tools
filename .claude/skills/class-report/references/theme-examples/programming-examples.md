# プログラミング系科目のテーマ例

テーマ候補生成時（Step 3・記述なし分岐）の参考例。科目の到達目標・授業計画に合わせてカスタマイズすること。

## テーマ例

### 調査・比較型

**テーマ: プログラミング言語の比較分析**
同一の課題を2つ以上の言語で実装し、記述量・可読性・実行速度等を比較する。言語の特徴を実践的に理解する。

**テーマ: バージョン管理システムの比較**
Git以外のバージョン管理システム（Mercurial、SVN等）を調査し、Gitとの違いを機能・ワークフロー・適用場面で比較する。

### 実装・制作型

**テーマ: CLIツールの開発と文書化**
日常的に使えるコマンドラインツールを開発し、READMEとユーザーマニュアルをMarkdownで作成する。設計判断とその理由も記述する。

**テーマ: Webアプリケーションの設計と実装**
小規模なWebアプリケーションを設計・実装し、要件定義書・設計書・実装手順書をMarkdownで作成する。

### 分析・考察型

**テーマ: オープンソースプロジェクトのコード分析**
GitHubで公開されているOSSプロジェクトを1つ選び、コード構成・Issue管理・PRフロー・コーディング規約を分析する。

**テーマ: 生成AI活用の効果測定**
プログラミング課題に対して生成AIあり/なしで取り組み、開発効率・コード品質・学習効果を比較考察する。

### 設計・提案型

**テーマ: チーム開発ワークフローの設計**
4〜5人のチームで開発する場合のブランチ戦略・Issue管理・レビュープロセスを設計し、根拠とともに提案する。
