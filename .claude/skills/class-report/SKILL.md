# レポート課題生成スキル

シラバスMarkdownの授業計画・到達目標・ルーブリック・評価割合をもとに、レポート・成果物課題の出題文と採点用ルーブリックを生成し、Markdownファイルとして出力するスキル。

## なぜこのスキルが必要か

高専の授業ではレポート・成果物の課題設計が頻繁に発生する。シラバスに到達目標・ルーブリック・授業計画が定義済みであるため、それらを入力として課題内容を具体化し、採点基準を明確にすることで、教員の課題設計負荷を削減できる。また、シラバスにレポートの詳細が記載されていない科目でも、到達目標やMCCカテゴリからテーマ候補を提案できる。

## 前提条件

- シラバスMarkdownファイルが `output/` 配下に存在すること（`/class-syllabus` スキルで事前に作成）
- 出力はすべてMarkdown形式

## デフォルト設定

| 項目 | 個人レポート | チームレポート |
|------|-------------|---------------|
| 配点 | 100点満点 | 100点満点（チーム共通点数） |
| ルーブリック段階 | 3段階（理想的/標準的/未到達） | 3段階（理想的/標準的/未到達） |
| 点数換算 | 理想的=80〜100%（標準点90%） / 標準的=60〜79%（標準点70%） / 未到達=0〜59%（標準点30%） | 同左 |

ユーザーが明示的に変更しない限り、上記のデフォルト値を適用する。

## 対話ルール

ユーザーへの質問は場面に応じて適切な方式を使い分ける:

- **自由入力が自然な場面**（ファイルパス、テーマの補足指示、修正内容など）: プレーンテキストで質問する
- **選択肢が明確な場面**（レポート選択、提出形態、テーマ候補など）: `AskUserQuestion` ツールを使う。自由入力は自動付与される "Other" で対応される

不明点や曖昧な点がある場合は、推測で進めずにユーザーに確認すること。

## 処理フロー

```
Step 0: シラバスMarkdownの選択と読み込み → レポートスケジュール自動抽出
Step 1: 作成するレポートの選択
Step 2: 提出設定の確認（個人/チーム・提出形式）
Step 3: テーマ・課題内容の確定（シラバス記述密度に応じた3分岐）
Step 4: 課題内容の詳細設計（要件・提出物・評価観点）
Step 5: ルーブリックの設計
Step 6: プレビューと修正ループ
Step 7: ファイル出力
Step 8: 完了レポート
```

### Step 0: シラバスMarkdownの選択と読み込み

プレーンテキストで質問する:
```
レポート課題を作成する科目のシラバスMarkdownファイルのパスを教えてください
（例: output/2026_アルゴリズム/26開講情報工学科3アルゴリズム.md）:
```

ファイルを読み込み、以下の情報を自動抽出して表示する:

```
=== シラバス情報 ===
科目名: [科目名]
担当教員: [教員名]
対象学年: [学年]
開講期: [期]
評価割合（レポート・課題）: [N]%

=== レポートスケジュール（「授業の進め方」から抽出）===
[レポート1  第N週提出  テーマ: ...]
[レポート2  第N週提出  テーマ: ...]
[最終レポート  第N週提出  テーマ: ...]

=== 到達目標 ===
[シラバスの到達目標を表示]

=== ルーブリック ===
| 評価観点 | 理想的 | 標準的 | 未到達 |
|----------|--------|--------|--------|
| ... | ... | ... | ... |
```

**抽出ロジック**:
- レポートスケジュールは「授業の進め方」セクションからパターンマッチで抽出する（「レポート」「課題」「成果物」「提出」等のキーワード）
- 授業計画の各週の内容に「レポート」「課題」「提出」等が含まれる場合も補完的に利用する
- 抽出に失敗した場合（レポートの記載がない等）は、プレーンテキストでユーザーに手動入力を求める:
  ```
  レポート情報を自動抽出できませんでした。以下の情報を教えてください:
  - レポートの名前（例: レポート1）:
  - 提出予定週:
  - テーマ（決まっている場合）:
  ```

**シラバス記述密度の自動判定**（内部処理、レポートごとに判定）:
- **詳細あり**: テーマ名と内容要件（提出物・分量・取り組み方等）が記述されている
- **概要のみ**: テーマ名または提出週のいずれかのみ記述されている
- **記述なし**: レポートへの言及が評価割合のみ、または全くない

この判定結果を Step 3 の分岐に使用する。

シラバスMarkdownが存在しない場合は `/class-syllabus` スキルでの事前作成を案内する。

### Step 1: 作成するレポートの選択

まず `AskUserQuestion` で作成範囲を確認する:
- options: 「すべてのレポートを作成する」「一部のレポートを選んで作成する」

**「すべてのレポートを作成する」** が選択された場合: Step 0 で抽出した全レポートを作成対象とし、Step 2 へ進む。

**「一部のレポートを選んで作成する」** が選択された場合: `AskUserQuestion` で個別に選択する:
- options: 抽出したレポート名リスト（例: 「レポート1（第8週提出・字句解析）」「レポート2（第11週提出・構文解析）」）
- multiSelect: true（複数選択可能）

**レポート名の表記ルール**: シラバスに「①②③」等の丸数字が使われている場合、「レポート1」「レポート2」のようにアラビア数字に変換して使用する。この名前はディレクトリ名・ファイル名で共通して使用する。

**複数レポートの処理**: 複数のレポートが選択された場合、Step 2〜Step 7 をレポートごとに繰り返す。各レポートの完了後に `AskUserQuestion` で「次のレポート（[レポート名]）に進む」「ここで終了する」を確認する。

### Step 2: 提出設定の確認

レポートごとに以下を確認する。

**提出形態**: `AskUserQuestion` で確認する:
- options: 「個人レポート」「チームレポート（チーム共通点数）」

「チームレポート」が選択された場合、追加で `AskUserQuestion` を使う:
- options: 「チーム内で役割分担あり（担当箇所を明記させる）」「チーム全員で同一内容を共同作成」

**提出形式**: `AskUserQuestion` で確認する:
- options: 「Markdown / テキスト」「PDF」「プログラム（ソースコード）」「プログラム + レポート」

提出期限はシラバスから自動抽出した値を提示し、プレーンテキストで「この期限で合っていますか？変更がある場合は入力してください」と確認する。

### Step 3: テーマ・課題内容の確定

Step 0 の記述密度判定に基づいて分岐する。

**[詳細あり] の場合**:

シラバス記載の内容を構造化して表示する:
```
=== シラバス記載の課題内容 ===
テーマ: [テーマ名]
概要: [シラバスに記載された説明]
提出週: 第[N]週
対象範囲: [対応する授業計画の週]
```

`AskUserQuestion` で確認する:
- options: 「この内容をベースに具体化する」「テーマ・内容を変更したい」

「変更したい」場合はプレーンテキストで変更内容を聞き、「概要のみ」ルートと同様にテーマを再設定する。

**[概要のみ] の場合**:

シラバス記載のテーマ名を提示し、到達目標・授業計画を参考に内容を補完した具体化案を1案生成して提示する:
```
=== テーマ補完案 ===
テーマ: [テーマ名]
補完した内容:
- 課題の方向性: [...]
- 想定する提出物: [...]
- 対応する到達目標: [...]
```

`AskUserQuestion` で確認する:
- options: 「この補完案で進む」「内容を修正したい」「テーマから変更したい」

**[記述なし] の場合**:

`references/report-design-guide.md` のテーマ候補生成ルールと `references/theme-examples/` の科目分野別テーマ例を参照し、シラバスの到達目標・授業計画・ルーブリックに基づいたテーマ候補を 3〜5個 生成して提示する:

```
=== テーマ候補 ===
以下のテーマ候補から選んでください:

1. [テーマ名]: [1〜2行の説明]
2. [テーマ名]: [1〜2行の説明]
3. [テーマ名]: [1〜2行の説明]
4. [テーマ名]: [1〜2行の説明]
5. [テーマ名]: [1〜2行の説明]
```

`AskUserQuestion` で選択する:
- options: テーマ候補の番号リスト + 「別のテーマを提案してほしい」

「別のテーマを提案してほしい」が選択された場合、プレーンテキストで方向性を聞き、再生成する。

テーマ選択後、プレーンテキストで確認: 「テーマの内容・方向性について追加の指示があれば教えてください（なければ「なし」と入力）」

### Step 4: 課題内容の詳細設計

Step 2・Step 3 の確定内容と `references/report-design-guide.md` の設計ルールに従い、以下を自動設計する:

1. **課題の目的**: 到達目標との対応を明示した1〜2文
2. **課題の要件**: 必須要件（提出物・分量・形式）と推奨要件（参考文献・図表等）
3. **取り組みの手順**: 学生が迷わないための段階的なガイダンス
4. **提出物の仕様**: ファイル形式・命名規則・提出先
5. **評価の観点（概要）**: 採点に使う評価観点の名前と配点の概要（詳細は Step 5 で設計）
6. **チーム固有の要件**（チームレポートの場合のみ）: 役割分担の記載要件、個人貢献の明示方法

設計した内容を表示し、`AskUserQuestion` で確認する:
- options: 「この内容で進む」「要件を修正したい」「手順を修正したい」

修正が選択された場合はプレーンテキストで具体的な変更内容を聞き、反映して再提示する。

### Step 5: ルーブリックの設計

`references/rubric-design-guide.md` に従い、採点ルーブリックを自動生成する。

**評価段階**: シラバスのルーブリックに合わせた3段階（理想的/標準的/未到達）を使用する。

**点数換算の基準**:
- 理想的: 観点配点の 80〜100%
- 標準的: 観点配点の 60〜79%
- 未到達: 観点配点の 0〜59%

**評価観点の設計方針**:
- シラバスのルーブリック評価観点がある場合はそれを優先的に採用する
- シラバスにない観点は課題の性質に応じて追加する（構成・論理展開、提出形式・体裁 等）
- チームレポートの場合はチーム協働に関する観点を追加する

ルーブリックを表形式で提示する:
```
=== ルーブリック案 ===
| 評価観点 | 理想的（80〜100%） | 標準的（60〜79%） | 未到達（0〜59%） | 配点 |
|----------|-------------------|-------------------|------------------|------|
| [観点1] | [記述] | [記述] | [記述] | [N]点 |
| [観点2] | [記述] | [記述] | [記述] | [N]点 |
| ... | ... | ... | ... | ... |
| **合計** | — | — | — | **100点** |
```

`AskUserQuestion` で確認する:
- options: 「このルーブリックで確定する」「評価観点を変更したい」「配点を変更したい」「段階の記述を修正したい」

変更が選択された場合はプレーンテキストで変更内容を聞き、修正して再提示する。配点の合計が100点になっていることを必ず検算する。

### Step 6: プレビューと修正ループ

出題文とルーブリックを `references/output-format.md` のフォーマットに従って一括でプレビュー表示する。

`AskUserQuestion` で確認する:
- options: 「確定する」「課題内容を修正したい」「ルーブリックを修正したい」「両方修正したい」

修正が選択された場合はプレーンテキストで修正指示を聞き、反映後に再プレビューする。確定するまで繰り返す。

### Step 7: ファイル出力

出力先ディレクトリ:
```bash
mkdir -p "output/[開講年度]_[教科名]/reports/[レポート名]/submissions"
```

例: 2026年度コンパイラのレポート1 → `output/2026_コンパイラ/reports/レポート1/`

出力するファイルとディレクトリ:
1. **`[レポート名]_課題.md`**: 学生向け出題文（`references/output-format.md` の課題文フォーマット参照）
2. **`[レポート名]_ルーブリック.md`**: 採点用ルーブリックシート（同ファイルのルーブリックフォーマット参照）
3. **`submissions/`**: 提出物配置用の空ディレクトリ。教員がMoodleからダウンロードした学生の提出物をここに配置し、`/class-report-check` スキルで採点する

### Step 8: 完了レポート

```
=== レポート課題生成完了 ===
科目名: [科目名]
担当教員: [教員名]

生成したレポート:
| レポート名 | 提出形態 | 提出期限 | 出力先 |
|-----------|---------|---------|--------|
| [名前] | [個人/チーム] | 第[N]週 | output/[YYYY]_[教科名]/reports/[名前]/ |

生成ファイル（各レポートにつき）:
- 課題文: [レポート名]_課題.md
- ルーブリック: [レポート名]_ルーブリック.md
- 提出物フォルダ: submissions/（採点時にここへ提出物を配置）
```
