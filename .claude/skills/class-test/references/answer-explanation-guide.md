# 模範解答・解説 記述ガイド

Step 7 で出力する解説Markdownの記述ルール。確認用Markdownとは別ファイルとして、復習資料・教員の参考資料として機能する。

## 解説Markdownの構成

```markdown
# [テスト名] 解説: [科目名]

## 基本情報
| 項目 | 内容 |
|------|------|
| 科目名 | [科目名] |
| テスト名 | [テスト名] |
| 試験範囲 | 第[X]〜[Y]週 |
| 問題数 | [N]問 / [M]点満点 |

---

## 問題1（多肢選択 / [N]点 / 基礎）

### 問題文
[問題文]

(A) [選択肢A]
(B) [選択肢B]
(C) [選択肢C]
(D) [選択肢D]

### 正解
**(B)**

### 解説
[詳細な解説。以下を含める:]
- なぜこの選択肢が正解か
- 各誤答選択肢がなぜ不正解か
- この問題が問う概念の背景知識

### 関連する到達目標
- [シラバスの到達目標から対応する項目を引用]

### 復習ポイント
- [この問題でつまずいた場合に復習すべき内容]
- [参照すべき教材・授業回]

---

## 問題2（数値問題 / [N]点 / 標準）

### 問題文
[問題文]

### 正解
**[数値]**

### 解説
[計算過程をステップバイステップで記述:]
1. [ステップ1]
2. [ステップ2]
3. [ステップ3]
→ 答え: [数値]

### 関連する到達目標
- [対応する到達目標]

### 復習ポイント
- [復習すべき内容]

---
```

## 記述ルール

### 解説の深さ

**基礎問題**: 概念の定義・基本原理を簡潔に説明する（3〜5文）

**標準問題**: 解法の手順を段階的に示し、なぜその手順を取るかを説明する（5〜10文）

**応用問題**: 問題の分析方法から解法の選択理由、計算過程まで詳細に説明する（10文以上）。類似問題への応用方法も言及する

### Moodleフィードバック（簡潔版）vs 解説Markdown（詳細版）

| 項目 | Moodleフィードバック | 解説Markdown |
|------|---------------------|-------------|
| 長さ | 1〜2文 | 段落〜複数段落 |
| 目的 | 試験中の即時フィードバック | 復習・学習資料 |
| 内容 | 正解/不正解の判定と最小限のヒント | 詳細な解説・関連知識・復習ポイント |
| 計算過程 | 結果のみ | ステップバイステップ |
| 誤答分析 | 「不正解です」程度 | なぜその誤答を選びやすいか |

### 各問題形式の解説パターン

**多肢選択**: 正解の理由 → 各誤答の分析（なぜ間違えやすいか）

**数値問題**: 計算過程をステップバイステップで記述 → 別解があれば言及

**穴埋めCloze**: 各空欄の正解とその理由 → コード全体の動作説明

**マッチング**: 各ペアの対応関係の根拠を説明 → 混同しやすいペアの注意点

**記述 Short answer**: 正解の定義・由来 → 表記のバリエーション

**計算 Calculated**: 一般式の導出 → 具体値での計算例 → 変数が変わっても手順は同じことの強調

### 「関連する到達目標」の記述

シラバスの到達目標からこの問題に対応する項目を引用する。複数の到達目標に関連する場合はすべて列挙する。

### 「復習ポイント」の記述

- この問題で問われている知識・スキルを明示する
- 関連する授業回（第N週）を参照先として示す
- 発展的な学習への導線があれば記載する
