# マッチング 作問例

情報工学科の授業（アルゴリズム、データ構造等）を想定したマッチング問題の例。

---

## 例1: 基礎（データ構造と特徴の対応）

### 問題文
各データ構造と、その主な特徴を正しく対応させよ。

| データ構造 | 特徴 |
|---|---|
| スタック | ? |
| キュー | ? |
| 配列 | ? |
| 連結リスト | ? |

選択肢:
- 後入れ先出し（LIFO）
- 先入れ先出し（FIFO）
- インデックスによるO(1)アクセス
- ノードのポインタで要素を連結
- キーによるO(1)検索 ← ダミー

### 正解
| データ構造 | 特徴 |
|---|---|
| スタック | 後入れ先出し（LIFO） |
| キュー | 先入れ先出し（FIFO） |
| 配列 | インデックスによるO(1)アクセス |
| 連結リスト | ノードのポインタで要素を連結 |

### フィードバック
各データ構造にはそれぞれ固有の特徴があります。スタックはLIFO、キューはFIFO、配列はインデックスアクセス、連結リストはポインタ結合が主な特徴です。ダミー選択肢の「キーによるO(1)検索」はハッシュテーブルの特徴です。

### 設計意図
- 難易度: 基礎
- ダミー選択肢1つで難易度を微調整
- 4ペア + 1ダミーの構成

## 例2: 標準（ソートアルゴリズムと安定性の対応）

### 問題文
各ソートアルゴリズムと、その安定性（同じ値の要素の順序が保持されるか）を正しく対応させよ。

| アルゴリズム | 安定性 |
|---|---|
| バブルソート | ? |
| クイックソート | ? |
| 挿入ソート | ? |
| ヒープソート | ? |

選択肢:
- 安定（同じ値の要素の相対順序が保持される）
- 不安定（同じ値の要素の相対順序が保持されない）
- 入力データにより安定性が変わる ← ダミー

### 正解
| アルゴリズム | 安定性 |
|---|---|
| バブルソート | 安定 |
| クイックソート | 不安定 |
| 挿入ソート | 安定 |
| ヒープソート | 不安定 |

### フィードバック
バブルソートと挿入ソートは隣接要素の比較・交換で動作するため安定です。クイックソートはピボットによる分割で遠距離交換が発生するため不安定です。ヒープソートもヒープ構築時に遠距離交換が発生するため不安定です。ソートの安定性は入力データではなくアルゴリズムの性質で決まります。

### 設計意図
- 難易度: 標準
- 4ペア + 1ダミーで構成。答え側は「安定」「不安定」の2種類だが、Moodleではドロップダウンで選択するため問題なく動作する
- ソートアルゴリズムの性質を正確に理解しているか確認

## 例3: 応用（探索アルゴリズムと適用条件の対応）

### 問題文
各探索アルゴリズムと、その適用に必要な前提条件を正しく対応させよ。

| 探索アルゴリズム | 前提条件 |
|---|---|
| 線形探索 | ? |
| 二分探索 | ? |
| ハッシュ探索 | ? |
| 二分探索木による探索 | ? |

選択肢:
- 特別な前提条件なし（任意の配列に適用可能）
- データが昇順または降順に整列されていること
- ハッシュ関数とハッシュテーブルが定義されていること
- データが木構造で、各ノードが左 < 親 < 右の関係を満たすこと
- データがヒープ条件を満たす完全二分木であること ← ダミー

### 正解
| 探索アルゴリズム | 前提条件 |
|---|---|
| 線形探索 | 特別な前提条件なし |
| 二分探索 | データが昇順または降順に整列されていること |
| ハッシュ探索 | ハッシュ関数とハッシュテーブルが定義されていること |
| 二分探索木による探索 | データが木構造で、各ノードが左 < 親 < 右の関係を満たすこと |

### フィードバック
各探索アルゴリズムにはそれぞれ適用可能な前提条件があります。線形探索は前提なし、二分探索は整列済み配列、ハッシュ探索はハッシュテーブル、BST探索はBST構造が必要です。ダミーの「ヒープ条件」はヒープソートやプライオリティキューに関連する条件です。

### 設計意図
- 難易度: 応用
- アルゴリズムの適用条件の正確な理解を問う
- 紛らわしいダミー（ヒープ条件）で判断力を確認
