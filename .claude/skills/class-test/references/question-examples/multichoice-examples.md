# 多肢選択 作問例

情報工学科の授業（アルゴリズム、データ構造等）を想定した多肢選択問題の例。

---

## 例1: 基礎（知識・理解）

### 問題文
スタック (Stack) のデータ操作方式として正しいものはどれか。

(A) FIFO（先入れ先出し）
(B) LIFO（後入れ先出し）
(C) ランダムアクセス
(D) 優先度順

### 正解
(B)

### フィードバック
- (A): FIFOはキュー (Queue) の特徴です。
- (B): 正解です。スタックは後に入れたデータが先に取り出されるLIFO構造です。
- (C): ランダムアクセスは配列の特徴です。
- (D): 優先度順は優先度キュー (Priority Queue) の特徴です。

### 設計意図
- 難易度: 基礎（ルーブリック「未到達」確認レベル）
- 観点: データ構造の基本特徴の識別
- ディストラクタ: 他のデータ構造の特徴を誤答に使い、混同を確認する

---

## 例2: 標準（応用）

### 問題文
要素数 n の整列済み配列に対して二分探索を行う場合、最悪の比較回数のオーダーとして正しいものはどれか。

(A) O(1)
(B) O(log n)
(C) O(n)
(D) O(n log n)

### 正解
(B)

### フィードバック
- (A): O(1) は最良の場合（1回目で見つかる場合）であり、最悪ではありません。
- (B): 正解です。二分探索は毎回探索範囲を半分にするため、最悪でも log₂n 回の比較で済みます。
- (C): O(n) は線形探索の最悪計算量です。
- (D): O(n log n) は効率的なソートアルゴリズム（クイックソート等）の平均計算量です。

### 設計意図
- 難易度: 標準（ルーブリック「標準的」レベル）
- 観点: 計算量の理解と各アルゴリズムとの対応
- ディストラクタ: よくある計算量を選択肢に配置し、正確な知識を確認

---

## 例3: 応用（分析・評価）

### 問題文
以下の条件でデータを管理する場合、最も適切なデータ構造はどれか。

**条件**: 要素の追加は末尾にのみ行い、要素の取り出しは先頭からのみ行う。要素数は最大1000件で、追加・取り出しともに O(1) で行いたい。

(A) 配列（固定長）
(B) スタック
(C) キュー（リングバッファ実装）
(D) 二分探索木

### 正解
(C)

### フィードバック
- (A): 配列では先頭からの取り出し時に要素のシフトが必要で O(n) になります。
- (B): スタックは末尾追加・末尾取り出し（LIFO）なので、先頭取り出しの条件に合いません。
- (C): 正解です。キューは先入れ先出し（FIFO）で、リングバッファ実装なら追加・取り出しともに O(1) です。最大1000件の固定長で実装できます。
- (D): 二分探索木の挿入・削除は O(log n) であり、O(1) の要件を満たしません。

### 設計意図
- 難易度: 応用（ルーブリック「理想的」レベル）
- 観点: データ構造の特性を理解した上での選択判断
- ディストラクタ: 条件を部分的に満たすが完全には合わない構造を選択肢にする
