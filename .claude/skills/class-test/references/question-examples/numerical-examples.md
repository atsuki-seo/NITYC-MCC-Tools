# 数値問題 作問例

情報工学科の授業（アルゴリズム、データ構造等）を想定した数値問題の例。

---

## 例1: 基礎（計算量の具体値）

### 問題文
要素数 8 の配列に対してバブルソートを実行する場合、最悪ケースでの比較回数を整数で答えよ。

### 正解
28（許容誤差: 0）

### 解説
バブルソートの最悪比較回数 = n(n-1)/2 = 8 × 7 / 2 = 28

### フィードバック
- 正解: 正解です。バブルソートの最悪比較回数は n(n-1)/2 で計算できます。
- 不正解: 不正解です。バブルソートの最悪比較回数は n(n-1)/2 = 8×7/2 = 28 です。

### 設計意図
- 難易度: 基礎
- 公式の適用力を問う
- tolerance = 0（整数解のため完全一致）

---

## 例2: 標準（アルゴリズムのトレース）

### 問題文
以下の配列に対して選択ソート（昇順）を実行する。2回目のパス完了後、配列の1番目から数えて3番目の要素の値を整数で答えよ。

初期配列: [5, 3, 8, 1, 4]

### 正解
8（許容誤差: 0）

### 解説
選択ソートのトレース:
1. 1回目のパス: 最小値 1 を先頭と交換 → [1, 3, 8, 5, 4]
2. 2回目のパス: 残り[3, 8, 5, 4]の最小値 3 は既に2番目 → [1, 3, 8, 5, 4]

2回目のパス完了後の配列: [1, 3, 8, 5, 4]
1番目から数えて3番目の要素: 8

### フィードバック
- 正解: 正解です。2回目のパス完了後の配列は [1, 3, 8, 5, 4] で、3番目の要素は 8 です。
- 不正解: 不正解です。選択ソートの各パスをトレースしてみましょう。

### 設計意図
- 難易度: 標準
- アルゴリズムの手順を正確にトレースする能力を問う
- AI耐性: 具体的な配列のトレース結果を問うため、一般論では解けない

---

## 例3: 応用（二分探索のステップ数）

### 問題文
以下の昇順整列済み配列に対して二分探索で値 15 を検索する場合、値が見つかるまでの比較回数を整数で答えよ。中央値の計算では端数を切り捨てる（floor）ものとする。

配列: [2, 5, 8, 10, 15, 20, 25, 30]

### 正解
3（許容誤差: 0）

### 解説
二分探索のトレース:
1. 1回目: 範囲[0..7], mid=3, arr[3]=10, 15>10 → 右半分[4..7]
2. 2回目: 範囲[4..7], mid=5, arr[5]=20, 15<20 → 左半分[4..4]
3. 3回目: 範囲[4..4], mid=4, arr[4]=15, 15==15 → 発見！

比較回数: 3回

### フィードバック
- 正解: 正解です。二分探索で3回の比較で値15が見つかります。
- 不正解: 不正解です。二分探索の各ステップをトレースしてみましょう。

### 設計意図
- 難易度: 応用
- 二分探索の手順を完全にトレースする能力を確認
- AI耐性: 具体的な配列・検索値での正確なトレースが必要
