# 確認用Markdown 出力フォーマット

Step 6 のプレビュー表示および Step 7 の確認用Markdown出力で使用するフォーマット。

## フォーマット

```markdown
# [テスト名]: [科目名]

## 基本情報
| 項目 | 内容 |
|------|------|
| 科目名 | [科目名] |
| 担当教員 | [教員名] |
| テスト種別 | 小テスト / 定期試験 |
| 実施予定 | 第[N]週 |
| 試験時間 | [N]分 |
| 問題数 | [N]問 |
| 配点 | [N]点満点 |
| 試験範囲 | 第[X]〜[Y]週（[単元名リスト]） |
| ポリシー | [持ち込み不可 / ノート持ち込み可 / AI・Web検索許可] |

## 問題構成サマリー
| 問題形式 | 問題数 | 配点 | 難易度 |
|----------|--------|------|--------|
| 多肢選択 | [N]問 | [M]点 | [基礎〜標準] |
| 数値問題 | [N]問 | [M]点 | [標準〜応用] |
| ... | ... | ... | ... |
| **合計** | **[N]問** | **[M]点** | — |

## 問題一覧

---

### 問題1（多肢選択 / [N]点 / 基礎）

[問題文]

(A) [選択肢A]
(B) [選択肢B]
(C) [選択肢C]
(D) [選択肢D]

> **正解**: ([X])
> **フィードバック**: [正解時のフィードバック]

---

### 問題2（数値問題 / [N]点 / 標準）

[問題文]

> **正解**: [数値]（許容誤差: [tolerance]）
> **フィードバック**: [フィードバック]

---

### 問題3（穴埋めCloze / [N]点 / 標準）

[問題文（穴埋め箇所を `[___]` で表示）]

> **正解**:
> - 空欄1: [正解]
> - 空欄2: [正解]
> **フィードバック**: [フィードバック]

---

### 問題4（マッチング / [N]点 / 標準）

[問題文]

| 問い | 答え |
|------|------|
| [左側1] | [右側1] |
| [左側2] | [右側2] |
| [左側3] | [右側3] |
| — | [ダミー] |

> **フィードバック**: [フィードバック]

---

### 問題5（記述 Short answer / [N]点 / 基礎）

[問題文]

> **正解**: [回答]（許容バリエーション: [別表記1], [別表記2]）
> **フィードバック**: [フィードバック]

---

### 問題6（計算 Calculated / [N]点 / 応用）

[問題文（変数を `{変数名}` で表示）]

> **計算式**: [式]
> **変数範囲**: [変数名] = [min]〜[max]
> **許容誤差**: [tolerance]
> **フィードバック**: [フィードバック]

---
```

## フォーマット規則

1. **問題番号**: 通し番号（問題1, 問題2, ...）
2. **見出し**: `### 問題N（形式 / 配点 / 難易度）` の形式で統一する
3. **問題間の区切り**: `---`（水平線）で区切る
4. **正解・フィードバック**: 引用ブロック（`>`）で表示する
5. **選択肢**: `(A)`, `(B)`, `(C)`, `(D)` のラベルで表示する
6. **穴埋め箇所**: 確認用Markdownでは `[___]` で表示し、正解は別途記載する
7. **Calculated変数**: `{変数名}` のまま表示し、変数範囲を明記する
8. **コードブロック**: コードを含む問題文では ``` で囲む
