# 学生向け事前通知テキスト 生成ガイド

テスト実施の1週間前〜数日前に学生へ展開するための通知テキストを生成するガイド。問題の正解や具体的な問題文は含めず、範囲・形式・準備方法を伝える。

## 通知テキストの構成

```markdown
## [科目名] [テスト名] のお知らせ

### 基本情報
- 実施日: 第[N]週の授業内
- 試験時間: [N]分
- 配点: [N]点
- 範囲: 第[X]〜[Y]週の内容
- 持ち込み/AI利用: [ポリシー]

### 試験範囲の内容
- 第[X]週: [授業内容]
- 第[X+1]週: [授業内容]
- ...

### 出題形式
| 問題形式 | 問題数 | 概要 |
|----------|--------|------|
| 多肢選択 | [N]問 | 概念理解・判断力を問う4択問題 |
| 数値問題 | [N]問 | 計算結果やトレース結果を数値で回答 |
| ... | ... | ... |

### 準備のアドバイス
- [範囲の内容に応じた具体的なアドバイス]

### おすすめの学習用プロンプト
以下のプロンプトをAIに投げて、テスト対策に活用してみてください:
1. 「[プロンプト例1]」
2. 「[プロンプト例2]」
3. ...
```

## 各セクションの記述ルール

### 基本情報
- テスト確定情報をそのまま記載する
- ポリシーは具体的に記載する（例: 「ノート類の持ち込み可、AI/Web検索の使用を許可」）

### 試験範囲の内容
- シラバスの授業計画から対象週の内容を箇条書きで列挙する
- 各週のキーワード・主要トピックを含める
- テスト実施週自体の内容（「小テスト①」等）は除外し、学習内容のみ記載する

### 出題形式
- 問題形式と問題数を表形式で提示する
- **具体的な問題内容は含めない**（「○○について出題する」のような予告はしない）
- 各形式の概要（何が問われるか）を一般的な表現で記載する

### 準備のアドバイス
- 範囲の内容に応じて **3〜5項目** の具体的なアドバイスを記載する
- 以下の観点を含める:
  - 理解すべき基本概念やキーワード
  - 手を動かして練習すべきこと（トレース、実装など）
  - 見落としがちなポイント
- 「〜を暗記しなさい」ではなく「〜を説明できるようにしておく」「〜をトレースできるようにしておく」のような表現を使う

### おすすめの学習用プロンプト

AI/Web検索が許可されている場合のみ記載する（クローズドブックの場合はこのセクションを省略する）。

**プロンプト例の設計方針**:
- テスト範囲の内容を効率的に復習・理解するためのプロンプトにする
- 問題の正解を直接得るものではなく、概念理解を深めるものにする
- 3〜5個のプロンプトを、異なる学習目的で用意する

## 情報開示の粒度

テストの測定妥当性を保つため、通知テキストに含める情報は以下の粒度に厳密に従うこと。

### 含めてよい情報

- **試験範囲**: 第X〜Y週の授業内容（シラバスで既に公開済みのため問題なし）
- **試験時間・配点・実施日時**: 基本情報として開示
- **ポリシー**: AI/Web検索許可の有無、持ち込み可否
- **問題形式の種類**: 多肢選択・数値問題・Cloze 等の **形式名** と **おおよその問題数**（レンジで表現: 「5〜7問程度」）
- **復習推奨ポイント**: 授業計画の範囲を要約した一般的な学習指針
- **当日の持ち物・注意事項**

### 含めてはならない情報

- **具体的な問題文・選択肢**
- **正解・模範解答**
- **重点キーワード**（「ここが出る」的な特定概念の指名）
  - ❌ 「ダイクストラ法の計算量が重要」
  - ⭕ 「アルゴリズムの計算量を評価できるようにしておく」
- **難易度分布の具体数値**（「基礎50%・標準30%・応用20%」のような開示）
- **Calculated問題のランダム変数の内容**
- **問題数の正確な値**（「多肢選択6問」のような固定的表現）
  - ❌ 「多肢選択: 6問」
  - ⭕ 「多肢選択: 5〜7問程度」

### 原則

「出題範囲の公開」は許容するが、「出題予告」は避ける。シラバス公開情報＋形式情報の範囲にとどめる。

## 注意事項

- **問題の正解や具体的な問題文は絶対に含めない**
- 通知テキストはテスト内容のネタバレにならないよう配慮する
- プロンプト例は、テストの問題に直接対応するものではなく、範囲全体の理解を助けるものにする
