# テスト問題生成スキル

シラバスMarkdownの授業計画・到達目標・試験範囲・ルーブリックをもとに、小テスト・定期試験の問題と模範解答を生成し、Moodle XML・確認用Markdown・解説を出力するスキル。

## なぜこのスキルが必要か

高専の授業では小テスト・定期試験の作成が定期的に発生する。シラバスに到達目標・ルーブリック・試験範囲が定義済みであるため、それらを入力として問題を自動設計・生成することで、教員の作問負荷を大幅に削減できる。Moodle XMLで出力することで、問題バンクへの即時インポートも可能になる。

## 前提条件

- シラバスMarkdownファイルが `output/` 配下に存在すること（`/syllabus` スキルで事前に作成）
- 出力する問題はすべて **Moodle 4.x の自動採点に対応する形式** に限定する（エッセイ形式は対象外）

## デフォルト設定

| 項目 | 小テスト | 定期試験 |
|------|----------|----------|
| 試験時間 | 15分 | 90分 |
| ポリシー | AI/Web検索許可 | AI/Web検索許可 |
| 配点 | 20点 | 100点 |

ユーザーが明示的に変更しない限り、上記のデフォルト値を適用する。シラバスの「授業の進め方」にポリシーの記載がある場合はそちらを優先する。

### 時間バッファルール

問題構成の想定解答時間の合計は、**試験時間の 3/4 以下** に収めること（試験時間の 1/4 以上をバッファとして確保する）。具体的な数値は `references/question-design-guide.md` の時間バッファルールを参照。Step 4 の時間配分チェックで必ず検証する。

## 対話ルール

ユーザーへの質問は場面に応じて適切な方式を使い分ける:

- **自由入力が自然な場面**（ファイルパス、問題の修正指示など）: プレーンテキストで質問する
- **選択肢が明確な場面**（テスト選択、構成確認、出力形式など）: `AskUserQuestion` ツールを使う。自由入力は自動付与される "Other" で対応される

不明点や曖昧な点がある場合は、推測で進めずにユーザーに確認すること。

---

## 処理フロー

```
Step 0: シラバスMarkdownの選択と読み込み → 試験スケジュール自動抽出
Step 1: 作成するテストの選択（小テスト①/②/③、期末試験 等）
Step 2: 試験情報の確認・補完（時間・ポリシー）
Step 3: 問題構成の自動設計（形式比率・問題数・難易度・配点）
Step 4: 問題構成案の提示 → 教員の合意
Step 5: 問題・模範解答の生成
Step 6: プレビュー・修正ループ
Step 7: ファイル出力（Moodle XML + 確認用Markdown + 解説）
Step 7.5: 学生向け事前通知テキストの生成（オプション）
Step 8: 完了レポート
```

### Step 0: シラバスMarkdownの選択と読み込み

プレーンテキストで質問する:
```
テスト問題を作成する科目のシラバスMarkdownファイルのパスを教えてください
（例: output/2026_アルゴリズム/26開講情報工学科3アルゴリズム.md）:
```

ファイルを読み込み、以下の情報を自動抽出して表示する:

```
=== シラバス情報 ===
科目名: [科目名]
担当教員: [教員名]
対象学年: [学年]
開講期: [期]

=== 試験スケジュール（「授業の進め方」から抽出）===
[小テスト①  第N週  範囲: 第X〜Y週]
[小テスト②  第N週  範囲: 第X〜Y週]
[期末試験    第N週  範囲: 全範囲]

=== 到達目標 ===
[シラバスの到達目標を表示]

=== ルーブリック ===
| 評価観点 | 理想的 | 標準的 | 未到達 |
|----------|--------|--------|--------|
| ... | ... | ... | ... |
```

**抽出ロジック**:
- 試験スケジュールは「授業の進め方」セクションからパターンマッチで抽出する（「小テスト」「試験」「範囲」等のキーワード）
- 授業計画の各週の内容に「小テスト」「試験」等が含まれる場合も補完的に利用する
- 抽出に失敗した場合（試験スケジュールが記載されていない等）は、プレーンテキストでユーザーに手動入力を求める:
  ```
  試験スケジュールを自動抽出できませんでした。以下の情報を教えてください:
  - テストの種類（小テスト / 定期試験）:
  - 実施予定週:
  - 試験範囲（第X〜Y週）:
  ```

シラバスMarkdownが存在しない場合は `/syllabus` スキルでの事前作成を案内する。

### Step 1: 作成するテストの選択

Step 0 で抽出した試験スケジュールをもとに、`AskUserQuestion` で作成対象を選択する:
- options: 抽出した試験名リスト（例: 「小テスト1（第5週・範囲: 第1〜4週）」「小テスト2（第8週・範囲: 第5〜7週）」「小テスト3（第10週・範囲: 第8〜9週）」「期末試験（第15週・全範囲）」）

**テスト名の表記ルール**: シラバスに「①②③」等の丸数字が使われている場合、テスト名は「小テスト1」「小テスト2」のようにアラビア数字に変換して使用する。この名前はディレクトリ名・XMLカテゴリ名・ファイル名で共通して使用する。

選択後、対象テストの範囲に対応する授業計画の内容を一覧表示する:

```
=== テスト範囲の授業内容 ===
テスト名: [テスト名]
範囲: 第X〜Y週

| 週 | 授業内容 | 到達目標 |
|----|----------|----------|
| X週 | ... | ... |
| Y週 | ... | ... |

=== 対応するルーブリック評価観点 ===
- [範囲内の到達目標に対応するルーブリック項目を表示]
```

### Step 2: 試験情報の確認・補完

デフォルト設定を提示し、`AskUserQuestion` で確認する:

**テスト時間・ポリシーの確認**:

```
デフォルト設定:
- 試験時間: [小テスト: 15分 / 定期試験: 90分]
- ポリシー: AI/Web検索許可
```

- options: 「デフォルト設定のまま進める」「時間を変更する」「ポリシーを変更する」「時間・ポリシーの両方を変更する」

シラバスの「授業の進め方」にポリシーの記載がある場合は、デフォルトの代わりにその内容を提示する。

「時間を変更する」が選択された場合、プレーンテキストで新しい時間を質問する。

**ポリシー変更時の選択肢**（変更が選択された場合のみ）:

`AskUserQuestion` で確認する:
- options: 「持ち込み不可・AI使用不可（クローズドブック）」「ノート類の持ち込み可」「AI/Web検索の使用を許可（AI耐性問題を増やす）」

ポリシーは Step 3 の問題構成設計に影響する:
- **クローズドブック**: 知識再現も含む幅広い形式
- **ノート持ち込み可**: 応用・分析レベルの問題を増やす
- **AI/Web検索許可**: Calculated問題（ランダム値）・具体的なトレース問題を増やし、AI耐性を高める

### Step 3: 問題構成の自動設計

`references/question-design-guide.md` の設計ロジックに従い、以下を自動生成する:

1. **問題形式の比率**: テスト種類・ポリシーに応じて自動決定
2. **各形式の問題数**: テスト時間から逆算
3. **難易度分布**: ルーブリックの3段階（理想的/標準的/未到達）をBloom's Taxonomyの認知レベルに対応付け
4. **配点設計**: 合計点を設定（小テスト: 20点、定期試験: 100点を標準）
5. **各問題に割り当てる単元・観点**: 範囲内の授業内容・到達目標から配分

### Step 4: 問題構成案の提示と確認

Step 3 で設計した構成案を表形式で提示する:

```
=== 問題構成案 ===
テスト名: [テスト名]
試験時間: [N]分
問題数: [N]問
配点: [N]点満点

| # | 問題形式 | 問題数 | 配点 | 対象単元 | 難易度 |
|---|----------|--------|------|----------|--------|
| 1 | 多肢選択 | 6問 | 各2点(12点) | 第1〜4週の概念理解 | 基礎〜標準 |
| 2 | 数値問題 | 2問 | 各3点(6点) | 計算量・トレース | 標準〜応用 |
| 3 | 穴埋めCloze | 1問 | 2点 | コード穴埋め | 標準 |
| ... | ... | ... | ... | ... | ... |
| 合計 | — | [N]問 | [M]点 | — | — |

=== 難易度分布 ===
- 基礎（知識・理解）: [N]問 ([X]%)
- 標準（応用）: [N]問 ([X]%)
- 応用（分析・評価）: [N]問 ([X]%)

=== 時間配分の目安 ===
| 問題形式 | 想定解答時間 |
|----------|-------------|
| ... | ...分 |
| **合計** | **[N]分** |
| **試験時間** | **[M]分** |
| **バッファ** | **[M-N]分（[X]%）** |

※ バッファは試験時間の25%以上を確保（デフォルト設定セクションの時間バッファルール参照）
```

`AskUserQuestion` で確認する:
- options: 「この構成で問題生成に進む」「問題数・形式比率を変更する」「難易度配分を変更する」「配点を変更する」

変更が選択された場合はプレーンテキストで具体的な変更内容を聞き、構成を修正して再度提示する。

### Step 5: 問題・模範解答の生成

確定した問題構成に基づき、`references/question-types-guide.md` の形式別作問ルールと `references/question-examples/` の作問例を参照しながら、問題を順に生成する。

各問題に以下を付与する:
- **問題文**: 明確で曖昧さのない記述
- **正解**: 模範解答
- **採点基準**: 配点・部分点の基準
- **フィードバック（簡潔版）**: Moodleで正解/不正解時に表示するテキスト
- **解説（詳細版）**: 復習資料として使える詳しい説明

**AI耐性の確保**:
- 単純な定義の再現ではなく、「〜の場合の計算量はどれか」「トレース結果を答えよ」など理解・応用レベルの問題を優先
- Calculated問題はランダム変数 `{a}`, `{b}` を使い、値がランダム化されるよう設計
- 具体的なコードや手順のトレースを含め、AIに単純に解かせにくい問題を意識する

### Step 6: プレビューと修正ループ

生成した全問題を `references/preview-format.md` のフォーマットに従ってMarkdown形式でプレビュー表示する。

`AskUserQuestion` で確認する:
- options: 「確定する」「特定の問題を差し替えたい」「難易度を調整したい」「問題を追加したい」「問題を削除したい」

修正が選択された場合:
- 「特定の問題を差し替えたい」→ プレーンテキストで問題番号と修正指示を聞き、該当問題のみ再生成
- 「難易度を調整したい」→ プレーンテキストで調整内容を聞き、該当問題を修正
- 「問題を追加したい」→ プレーンテキストで追加内容を聞き、新しい問題を生成
- 「問題を削除したい」→ プレーンテキストで削除する問題番号を聞き、削除

修正後は再度プレビューを表示し、確定するまで繰り返す。

### Step 7: ファイル出力

`AskUserQuestion` で出力形式を確認する:
- options: 「3ファイルすべて出力（推奨）」「確認用Markdown + 解説のみ（XML不要）」「Moodle XMLのみ」

出力先ディレクトリ:
```bash
mkdir -p "output/[開講年度]_[教科名]/tests/[テスト名]"
```

例: 2026年度アルゴリズムの小テスト① → `output/2026_アルゴリズム/tests/小テスト1/`

出力する3ファイル:
1. **`[テスト名]_確認用.md`**: Step 6 のプレビュー内容をそのまま保存（`references/preview-format.md` 参照）
2. **`[テスト名]_解説.md`**: 各問題に詳細な解説・復習ポイントを追加したもの（`references/answer-explanation-guide.md` 参照）
3. **`[テスト名]_moodle.xml`**: Moodle問題バンクインポート用XML（`references/moodle-xml-format.md` 参照）

Moodle XMLの生成時:
- カテゴリは `$course$/[テスト名]` に設定する（コース自体が科目に対応するため、科目名の階層は不要）
- 問題文にHTMLやLaTeX数式を含む場合は `<![CDATA[...]]>` で囲む
- エンコーディングは UTF-8、BOMなし
- 生成後、XMLの構文妥当性を確認する（タグの開閉、必須属性の有無）

### Step 7.5: 学生向け事前通知テキストの生成

Step 7 のファイル出力後、`AskUserQuestion` で事前通知テキストの生成を提案する:
- options: 「学生向け事前通知テキストを生成する」「不要（スキップ）」

「生成する」が選択された場合、`references/student-notice-guide.md` のガイドに従い、**学生がテスト1週間前〜数日前に受け取る通知テキスト** を生成する。通知テキストには問題の正解や具体的な問題文は含めない。

`AskUserQuestion` で通知テキストの確認を取る:
- options: 「この内容で確定する」「修正したい」

確定後、テスト出力先と同じディレクトリに `[テスト名]_事前通知.md` として保存する。

### Step 8: 完了レポート

```
=== テスト問題生成完了 ===
科目名: [科目名]
担当教員: [教員名]
テスト名: [テスト名]
試験時間: [N]分
問題数: [N]問 / [M]点満点
出力先: output/[YYYY]_[教科名]/tests/[テスト名]/

生成ファイル:
- 確認用Markdown: [テスト名]_確認用.md
- 解説: [テスト名]_解説.md
- Moodle XML: [テスト名]_moodle.xml
- 事前通知: [テスト名]_事前通知.md（生成した場合）

問題構成:
| 問題形式 | 問題数 | 配点 |
|----------|--------|------|
| ... | ... | ... |
| 合計 | [N]問 | [M]点 |

難易度分布:
- 基礎: [N]問 ([X]%)
- 標準: [N]問 ([X]%)
- 応用: [N]問 ([X]%)
```

## 対応する問題形式

Moodle 4.x 自動採点対応の以下6形式をサポートする: multichoice, numerical, cloze, matching, shortanswer, calculated。各形式の詳細な作問ルール・用途・品質基準は `references/question-types-guide.md` を参照。
