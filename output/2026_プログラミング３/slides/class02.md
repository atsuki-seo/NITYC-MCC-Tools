# Markdown とは

- プレーンテキストで構造化文書を書く軽量マークアップ
- `.md` 拡張子。VS Code でプレビュー可能
- HTML や PDF に容易に変換できる
- 技術ドキュメント・README・Issue/PR で標準的に使われる

---

<!--
class: flex-layout natural-height
-->

# 5つの基本記法

| 記法 | 書き方 | 用途 |
|------|--------|------|
| 見出し | `# H1` `## H2` `### H3` | 文書の章立て |
| 箇条書き | `- 項目` | 並列情報の列挙 |
| 表 | `| 列 | 列 |` | 比較・対応関係 |
| リンク | `[文字](URL)` | 出典・参照 |
| 画像 | `![alt](path)` | 図・写真の挿入 |

---

<!--
class: flex-layout natural-height
-->

# 読み手を意識するコツ

1. **見出しの階層を崩さない** — h1 の下は h2、h2 の下は h3
2. **1段落1主題** — 伝えたいことを1つに絞る
3. **表は列の意味を揃える** — 数値列と文字列列を混在させない
4. **リンクには出典の意味を込める** — 裏付けとなる情報元を示す
5. **画像には alt テキスト** — 説明文を書く

# 必須要件

- 5種類のMarkdown記法（見出し・箇条書き・表・リンク・画像）をすべて使用
- 構成要素:
  - 店舗概要
  - メニューと価格帯の分析（表）
  - 競合他社との比較（表）
  - 独自の魅力に関する考察
  - 参考文献リンク
- 解説と分析を区別して記述
- 出典を明記、巻末にAI活用プロンプト記録
