# テーマ選定の指針

- 7週間で **動く** ところまで持っていける範囲
- 全員が手を動かせる規模（機能を分担できる）
- 身近な課題・自分たちが使いたいもの
- 外部 API や DB を活用するとリアリティが出る

> 教員の確認を得てから着手すること

# 役割分担の観点

- 全員が **コード作成 / Issue / PR のいずれかに関与** すること（必須）
- 役割例:
  - プロジェクトリード（工程管理・リポジトリ設定）
  - 要件・設計ドキュメント担当
  - フロントエンド / バックエンド / DB 担当
  - レビュー担当・テスト担当
- 1人1役に固定せず、必要に応じて入れ替える

# 最終レポート: 基本情報

- **提出期限**: 第14週 / **提出形態**: チーム
- **提出物**: GitHub リポジトリ + 仕様書3点（PDF）+ 振り返り報告書 + AI 活用記録
- **提出先**: Moodle にディレクトリを ZIP 圧縮してアップロード
- **ディレクトリ名**: `チーム名_メンバー氏名1_メンバー氏名2_..`

class: flex-layout natural-height

# 必須要件（抜粋）

- GitHub 公開リポジトリの作成（または教員招待）
- 全メンバーが code / Issue / PR のいずれかに関与
- **仕様書3点**: 要件定義 / 基本設計 / 詳細設計
- **PR 経由のマージ**（main 直 push 禁止）、最低 **3件以上のPR**
- **Issue 10件以上** で作業管理
- README.md に動作手順
- KPT 振り返り + AI 活用プロンプト記録
