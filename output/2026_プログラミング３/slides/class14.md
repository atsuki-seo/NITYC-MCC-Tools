# 仕様書の PDF 化

- Markdown → PDF 変換は以下が利用可能:
  - VS Code 拡張「Markdown PDF」
  - `pandoc`（コマンドライン）
  - Marp（プレゼン用だが PDF 出力可）
- **3点セットとも PDF 化** してリポジトリ内 `docs/` に配置
- PDF はチームの提出ディレクトリにも同じものを入れる

# 振り返り報告書の構成

| 項目 | 内容 |
|------|------|
| メンバー氏名・学籍番号 | 冒頭に全員分を記載 |
| 担当箇所・貢献内容 | 各メンバーの役割 |
| KPT 集約（9〜14週） | Keep/Problem/Try の総まとめ |
| うまくいったこと | 具体例を数値・エピソードで |
| 課題と改善案 | 次回やるならどう変えるか |
| AI 活用を通じた学び | プロセスへの影響 |

# 振り返りを形骸化させないコツ

<div class="summary-positive">

## Tips

- 「コミュニケーションが大事」で終わらせず **具体的な場面** を書く
- 失敗も積極的に記述する（減点しない、むしろ学びを評価）
- 数値で示せるところは示す（Issue 数・PR 数・レビュー数）
- 個人 KPT ではなく **チーム KPT** として議論した上でまとめる

</div>
