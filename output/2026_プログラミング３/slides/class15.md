# 準備のポイント

- **git 基本コマンド** の目的と使いどころを自分の言葉で説明できること
  - `init / add / commit / branch / switch / merge / push / pull / clone / rebase / status / log`
- 実際に「ブランチ作成 → commit → push → PR → merge → ブランチ削除」を手でトレース
- コンフリクトの **正しい解消手順** と破壊的操作の違いを整理
- Markdown 記法を「どの場面で使うのが適切か」で説明できるように

# 学習用プロンプト例

AI に投げて理解を補強する（回答は授業教材と照合）:

4. Markdown 主要記法の適切な使い分け
