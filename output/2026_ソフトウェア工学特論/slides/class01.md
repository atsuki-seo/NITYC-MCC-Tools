# 授業の進め方

- 1回 = 講義（最大30分）＋演習（60分）の構成
- 教材配布は Moodle（<https://moodle2026.center.yuge.ac.jp>）
- 毎週の提出物あり
  - 第2・3・15週：レポート提出
  - 第9〜14週：KPT形式の自己評価（毎週）
- すべての提出物で **生成AI活用のプロンプト記録** が必須
- Markdownレポートは `.md` / `.txt` 以外は受理しない

# 評価割合

| 評価方法 | 配分 |
|---------|------|
| 相互評価 | 50% |
| ポートフォリオ | 50% |
| **合計** | **100%** |

- 素点（100点）× 配分で最終成績を算出
- 試験・定期試験は実施しない
- チーム運営とレポート類での継続評価を重視

---

<!--
class: flex-layout natural-height
-->

# 採点時の重要ポイント

- **チーム開発期間のKPT自己評価**：1点×回数（提出漏れ・空白・意味のない文章は0点）
- **全メンバーが以下のいずれかに関わること**：ソースコードの作成 / Issue作成 / PR作成
- **生成AIのプロンプト共有**は全提出物で必須
- 学修単位：1単位あたり30時間の自学自習が必要

結果だけでなく **改善点を出せるか** を評価する。
