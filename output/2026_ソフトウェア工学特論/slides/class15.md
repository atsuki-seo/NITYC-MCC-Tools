# KPT総括の進め方

1. 第9〜14週分の自分のKPTを **時系列で並べる**
2. Keep / Problem を **上位概念** で束ねる
3. 重複排除後、**各3項目以上** に整理
4. 具体的な **週・場面・当事者** を添える
5. 「良かった」「大変だった」で終わらせない

# Problemの掘り下げ

- 表層の不満で止めない
- 原因を **ワークフロー / 役割分担 / コミュニケーション** の層で分析
- 「時間が足りなかった」→ **なぜ足りなかったか** を1段階掘る
- 根本原因が見えると Try が具体化する

例：「PRレビューが滞った」→「Issue粒度が大きすぎた」→「分解基準が未定義だった」

---

<!--
class: flex-layout natural-height
-->

# Try = 動詞始まりの行動

| ✗ 精神論 | ○ 行動レベル |
|---------|------------|
| 頑張る | Issueは1PRマージ粒度まで分解する |
| 気をつける | PRテンプレートを導入して完了条件を必須化 |
| もっと早く動く | Issue作成を毎週月曜の30分で固定化 |

**誰が・いつ・何を** するかまで落とす。

---

<!--
class: flex-layout natural-height
-->

# Problem-Try対応表

```markdown
| Problem | Try | 優先度 | 着手時期 |
|---------|-----|--------|---------|
| Issue粒度が大きすぎた | 分解基準を事前合意 | 高 | 次回第1週 |
| レビューコメントが形式的 | レビュー観点チェックリスト作成 | 中 | 次回第2週 |
| AI出力をそのまま採用した | 批判評価フォーマット導入 | 中 | 次回第3週 |
```

優先度と着手時期が **評価観点「優先度設定と実行計画」** の核。
