# 応用記法一覧

| 記法 | 書き方 | 用途 |
|------|--------|------|
| 引用 | `> 文章` | 原典・注釈 |
| 強調 | `**太字**` `*斜体*` | 重要箇所の明示 |
| コード | `` `code` `` / ``` | 命令・識別子 |
| 水平線 | `---` | 区切り |
| 目次 | `## 目次` + リスト | 全体構造の予告 |

前回の5記法と組み合わせて、情報の重要度を表現する。

# 引用と強調の使い分け

```markdown
> 公式サイトによれば、2024年時点の店舗数は
> 全国1,234店舗である [^1]

重要な分量は **15g** を守る。
```

- 引用（`>`）は **他者の記述をそのまま取り込む** ときのみ
- 強調（`**`）は読み手が見落としてはいけない数値・キーワード
- どちらも **濫用すると効果が失われる**

# 曖昧な表現 → 定量表現

| 曖昧 | 定量 |
|------|------|
| 少々 | 1g（小さじ1/5） |
| 適量 | 大さじ1 |
| 中火でしばらく | 中火で3分 |
| 小さく切る | 2cm角に切る |
| きつね色になるまで | 表面が茶色になるまで（約4分） |

技術文書の要諦：**他人が同じ結果を再現できる**こと。

---

<!--
class: flex-layout natural-height
-->

# なぜ定量表現が重要か

- 料理も技術文書も **再現性** が本質
- 書き手の経験に依存する表現は第三者に伝わらない
- 数値化 → 工程の妥当性が検証可能になる
- 「誰が読んでも同じ結果に到達できる」ことが技術者の責務
- 次週から始まるgitも同じ：**誰の環境でも同じ動作** が前提

# 必須セクションと定量要件

- 含めるセクション：概要 / 材料 / 調理器具 / 下準備 / 調理手順 / コツ / 完成イメージ
- 材料は **表形式**（分量・人数分・代替材料）
- 手順は **番号付きリスト**（各手順に所要時間）
- **「少々」「適量」禁止** → g / ml / 大さじ / 小さじで記載
- 加熱時間・火力・切断サイズは数値で明示
