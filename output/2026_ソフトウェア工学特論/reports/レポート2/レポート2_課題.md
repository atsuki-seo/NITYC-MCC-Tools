## 課題の目的

Markdown を用いた構造化文書と定量的な表現方法を応用し、「他人が再現できる技術手順書」を作成する能力を養うことを目的とする。料理という身近な題材を通じて、手順の過不足・曖昧さ・前提条件の欠落といった技術文書で起こりがちな問題を、自ら発見・修正するトレーニングを行う。

## 課題の内容

### テーマ

**任意の料理の手順書**

自分で作れる料理（調理経験のあるもの）を1品選び、その料理を**初めて作る読者でも再現できるレベル**の手順書としてまとめる。レシピサイトのコピーではなく、自分の手順・分量・コツに基づいて記述すること。

### 要件

**必須要件**

- 以下のセクションをすべて含めること
  - 料理の概要（どんな料理か・難易度・所要時間）
  - 材料（表形式で、分量・人数分・代替材料を記載）
  - 調理器具（箇条書き）
  - 下準備（番号付きリスト）
  - 調理手順（番号付きリスト・各手順に所要時間の目安を付す）
  - コツ・注意点（箇条書き）
  - 完成イメージ（画像）
- **定量的な表現** を徹底すること
  - 分量は「少々」「適量」などではなく、計量単位（g / ml / 大さじ / 小さじ）で記載
  - 加熱時間・火力を具体的に記述（例: 「中火で3分」）
  - 切り方のサイズを数値で明示（例: 「2cm 角」）
- 文字数: 1500〜3000字（手順表・材料表を除く）
- 生成AI を利用した場合は、投入したプロンプトをレポート末尾の付録に記載すること（`## 付録: 生成AI活用記録`）

**推奨要件**

- 工程ごとの写真または図解を挿入する
- トラブル対処（よくある失敗と対処法）のセクションを追加する
- 材料の代替案や、アレルギー対応の注釈を付す

### 取り組みの手順

1. 題材となる料理を選定し、自分の手順を書き出す（memo レベルで構わない）
2. 材料・分量を計量しながら実際に再現し、曖昧だった手順を明確化する
3. 文書構造（セクション・見出し）を設計する
4. 材料表・手順リストを Markdown で記述する。定量的な表現に書き換える
5. 写真・画像を挿入し、Markdown プレビューで体裁を確認する
6. **他人が読んで再現できるか** を自問しながら見直し、曖昧な箇所を具体化する
7. 生成AI を活用した場合はプロンプト記録を付録に追記する
