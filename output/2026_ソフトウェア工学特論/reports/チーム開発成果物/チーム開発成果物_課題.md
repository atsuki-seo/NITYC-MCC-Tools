## 基本情報

| 項目 | 内容 |
|------|------|

| 担当教員 | 瀬尾 敦生 |

| レポート名 | チーム開発成果物 |
| 提出期限 | 第14週 |

## 課題の目的

チームで実システムの開発プロジェクトを遂行し、「git/GitHub を用いたチーム開発ワークフローの実践」と「プロジェクト管理手法に基づくチーム運営」を統合的に経験する。成果物として、動作するソフトウェアと、そこに至るまでの設計・運用の記録（仕様書・AI活用記録）をチームで提出することで、到達目標2・3の達成度を評価する。

## 課題の内容

### テーマ

**チームで設計・実装・運用するソフトウェア開発プロジェクト**

第8週で決定したテーマに基づき、チームでシステムを要件定義から実装・テストまで遂行する。テーマは自由だが、GitHub 上で Issue・Pull Request・コードレビューが成立する規模（全メンバーが最低1本以上の PR をマージする）であること。

### 要件

**必須要件**

- **GitHub リポジトリ**
  - チーム専用の GitHub リポジトリを作成し、全メンバーが Collaborator として参加
  - 第8週で決定したブランチ戦略（例: GitHub Flow / Git Flow）に従って運用
  - **全メンバーが最低1本以上の Pull Request をマージすること**
  - Issue を用いてタスク管理を行うこと（最終的に10件以上の Issue が存在すること）
  - PR には最低1名のレビュアーによるレビューコメントが付いていること
- **仕様書 PDF**: チームで1本作成し、以下を含める
  - システム概要・目的
  - 要件定義書（機能要件・非機能要件）
  - 基本設計（画面設計・DB 設計または外部仕様・システム構成図）
  - 詳細設計（主要モジュールの処理フロー）
  - チーム運用記録（ブランチ戦略・役割分担・開発期間中の意思決定ログ）
- **AI 活用記録**: チームで1本作成し、以下を含める
  - 使用した生成AIツール名
  - 主要な場面で投入したプロンプトとその出力
  - AI 出力に対するチームでの判断（採用／修正／却下の理由）
- **メンバー貢献記録**: 仕様書または別ファイルに、各メンバーの担当箇所・マージした PR 番号・主な貢献内容を明記
- 全メンバーが以下のいずれかに必ず関わること: ソースコードの作成 / Issue の作成 / PR の作成

**推奨要件**

- README.md でプロジェクトの概要・起動方法・使用技術を説明する
- CI（GitHub Actions 等）を導入し、PR 時の自動テストを運用する
- テストコードを含める
- コミットメッセージの規約（例: Conventional Commits）を定める

### 取り組みの手順

1. （第8週）テーマを決定し、GitHub リポジトリを作成、ブランチ戦略を決定する
2. （第9週）要件定義を行い、Issue に落とし込む
3. （第10〜11週）基本設計・詳細設計を作成し、設計ベースの Issue を追加する
4. （第12〜13週）実装を分担で進める。PR を作り、相互にコードレビューする
5. （第13週）動作検証・テストを行う。仕様書と AI 活用記録を並行して整備する
6. （第14週）成果物を最終調整し、リポジトリ URL・仕様書 PDF・AI 活用記録を提出する

## 提出物の仕様

- **ファイル形式**:
  - GitHub リポジトリ URL（README にアクセス権を明記）
  - 仕様書: PDF（`specification.pdf`）
  - AI 活用記録: PDF または Markdown（`ai_log.pdf` または `ai_log.md`）
- **提出先**: Moodle（https://moodle2026.center.yuge.ac.jp）
- **チーム提出時の配置**: 下記「チーム提出に関する注意事項」を参照
- **分量の目安**: 仕様書は A4 10〜20ページ、AI 活用記録は A4 3〜10ページ

## チーム提出に関する注意事項

- **提出ディレクトリ名**: `チーム名_メンバー氏名1_メンバー氏名2_メンバー氏名3`（例: `チームA_山田_佐藤_鈴木`）
- **提出ファイル配置**: 上記ディレクトリ内に以下を配置する
  - `README.md`（リポジトリURL・メンバー一覧・担当箇所を記載）
  - `specification.pdf`（仕様書）
  - `ai_log.pdf` または `ai_log.md`（AI 活用記録）
- 仕様書冒頭にチームメンバー全員の氏名・学籍番号を記載すること
- 各メンバーの担当箇所（設計担当・実装担当・レビュー担当等）とマージした PR 番号を明記すること
- チーム共通点数として採点するため、メンバーによる貢献差が大きい場合は「チーム運用と役割分担」観点で減点対象となる
