# 対話規約・修正ループ作法

ユーザーとの対話方式（質問の出し方）と、生成物のプレビュー・修正ループの**共有契約**。`class-` プレフィックスの全スキルで一貫した対話体験を提供するための正本。

このファイルは「質問方式の使い分け」「修正ループのパターン」を一元管理する。各スキルのSKILL.md冒頭に同じ「対話ルール」を二重記述せず、必ず本ファイルを参照すること。

## 参照元スキル

- `class-syllabus`
- `class-syllabus-parse`
- `class-test`
- `class-report`
- `class-report-check`
- `class-load`
- `class-slides`

## 1. 質問方式の使い分け

ユーザーへの質問は、場面に応じて以下の2方式を使い分ける。

### 1.1 プレーンテキストで質問する場面

**自由入力が自然な場面**:

- 教員名・科目名・ファイルパス・教科書/教材URL等の固有値
- 到達目標・ルーブリック等の長文入力
- 修正指示の具体的な内容（「問題3を計算問題に変えて」等）
- 課題のテーマ補足・チームメンバー情報
- 数値の自由入力（試験時間の任意指定など）

書式例:

```
担当教員のお名前を教えてください:
```

```
シラバスMarkdownファイルのパスを教えてください
（例: output/2026_アルゴリズム/26開講情報工学科3アルゴリズム.md）:
```

### 1.2 AskUserQuestion を使う場面

**選択肢が明確な場面**:

- 提示した複数案からの選択（「定期試験50分 / 90分」等）
- フロー分岐の選択（「全件作成 / 一部選択」等）
- 確認/修正/中断の3択（「確定する / 修正したい / 中断する」等）
- 既存ファイル検出時の上書き判断（`specs/file-output-policy.md` 参照）

ツール側仕様:

- 自由入力は自動付与される **"Other"** で対応される。選択肢に「自由入力」を明示的に含める必要はない
- multiSelect が必要な場面では `multiSelect: true` を指定する

### 1.3 不明点・曖昧な点

不明点や曖昧な点がある場合は、推測で進めずに必ずユーザーに確認する。プレーンテキストか AskUserQuestion かは上記基準で判断する。

## 2. プレビュー・修正ループの作法

生成物（シラバス・テスト・レポート課題・採点結果・スライド等）を出力する前には、必ずプレビューと修正ループを実行する。

### 2.1 標準パターン

```
1. 生成物の全内容をMarkdown形式でプレビュー表示
2. AskUserQuestion で確認
   - options: 「確定する」「[項目A]を修正したい」「[項目B]を修正したい」...
3. 修正が選択された場合:
   a. プレーンテキストで具体的な修正指示を聞く
   b. 該当部分のみ反映
   c. 再度プレビュー表示
   d. 再度 AskUserQuestion で確認
4. 「確定する」が選ばれるまで 3 を繰り返す
```

### 2.2 選択肢の構成

確認の AskUserQuestion 選択肢は以下の構成を標準とする:

- 必ず「確定する（または同等の進行選択肢）」を含める
- 修正対象が複数カテゴリある場合、カテゴリ別の修正選択肢を提示する（例: 「課題内容を修正したい」「ルーブリックを修正したい」「両方修正したい」）
- 中断・スキップが意味を持つフロー（複数件処理など）では「このスキップして次へ」「すべて中断」も含める

### 2.3 部分修正の原則

- 修正は**該当部分のみ差し替える**。全体を再生成しない
- 修正後の再プレビューでは、変更箇所が明示的に分かるように表示する（例: 「修正済み: #3, #7」と明記）
- 修正回数に上限は設けない（ユーザー判断で確定する）

### 2.4 検算が必要な修正

合計点・配点・評価割合など、合計値が固定値（100点・100%等）でなければならない項目を修正した場合、修正反映後に必ず検算する。検算で値が合わない場合はユーザーに伝え、再修正を求める。

## 3. 進捗・完了報告

### 3.1 処理中の進捗

長時間かかる処理（一括採点・複数テスト生成など）では、進捗を逐次表示する。

```
採点進捗: [N]/[M] 完了 — 現在: [学籍番号] [氏名]
```

### 3.2 完了報告

スキル完了時には、以下を含む完了レポートを表示する:

- 処理対象（科目名・教員名等）
- 生成物の一覧（ファイルパス）
- 件数・状態（完了 / スキップ / 未完了等）

完了レポートのフォーマットは各スキルの SKILL.md 末尾を参照（スキル固有の表示項目があるため、ここでは形式の統一までは強制しない）。

---

## 4. エラー・中断時の作法

- スクリプトやツールがエラーを返した場合、エラー内容をユーザーに提示し、対応（再試行・パス再入力・スキップ等）を確認する
- ユーザー判断で処理を中断する場合、ここまでに生成・保存済みのファイルはそのまま残し、未完了分を完了レポートで明示する
- 「すべて中断して終了」が選ばれた場合、追加の確認なしに即座に終了する
