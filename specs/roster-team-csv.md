# 履修者名簿・チーム構成CSV スキーマ

履修者名簿・チーム構成CSVの**共有契約**。`class-report` がチーム構成として収集し、`class-report-check` が読み取るCSVファイルのフォーマットを統一するための正本。

このファイルは「カラム名の揺らぎ対応」「文字コード判定」「未提出判定の扱い」を一元管理する。各スキルのリファレンスに同じ定義を二重記述せず、必ず本ファイルを参照すること。

## 参照元スキル

- `class-report`: チームレポートの提出設定でチーム構成CSVを受理
- `class-report-check`: 履修者名簿（採点対象の確定・未提出判定）とチーム構成（チーム単位採点）を読み取り

---

## 1. 基本仕様

| 項目 | 仕様 |
|------|------|
| 形式 | CSV |
| 文字コード | UTF-8（BOM有無どちらも可）/ Shift-JIS を自動判定して受理 |
| ヘッダー行 | 1行目をヘッダーとみなす |
| 区切り文字 | カンマ (,) |

### 1.1 文字コード判定

以下の順で試行する:

1. UTF-8 BOM付き
2. UTF-8
3. Shift-JIS（Windows で Excel から CSV 保存した場合に多い）
4. いずれも失敗した場合はユーザーにエンコーディングを確認する

---

## 2. 履修者名簿の列構成

### 2.1 必須列

- **学籍番号**: 学生の学籍番号
- **氏名**: 学生の氏名

その他の列（メールアドレス・所属・学年等）があっても構わないが、採点処理では使用しない。

### 2.2 カラム名の揺らぎ対応

以下のいずれかのヘッダー名にマッチすれば、対応する論理カラムとして認識する:

| 論理カラム | 受理するヘッダー名 |
|-----------|-----------------|
| 学籍番号 | `学籍番号`, `学生番号`, `番号`, `No.`, `No`, `ID`, `学生ID` |
| 氏名 | `氏名`, `名前`, `学生氏名`, `学生名`, `Name` |

### 2.3 自動マッピングの流れ

1. CSV を読み込み、1行目のヘッダー値を取得する
2. 上記揺らぎパターンでヘッダーを論理カラムにマッピングする
3. **成功時**: 「学籍番号列: [ヘッダー名]、氏名列: [ヘッダー名] でマッピングしました」と表示し、`AskUserQuestion` で「このまま進める」「マッピングを修正したい」を確認する
4. **失敗時**: `AskUserQuestion` で各論理カラムに対応するヘッダーを選択させる（選択肢は全ヘッダー名）

---

## 3. チーム構成ファイルの列構成

### 3.1 必須列

- **チーム名**: チームの識別子
- **学籍番号**: メンバーの学籍番号
- **氏名**: メンバーの氏名

1学生1行で、同じチーム名を持つ行のメンバーを同一チームに集約する。

### 3.2 カラム名の揺らぎ対応

履修者名簿の揺らぎ対応に加えて以下を認識する:

| 論理カラム | 受理するヘッダー名 |
|-----------|-----------------|
| チーム名 | `チーム名`, `班`, `班名`, `グループ`, `グループ名`, `Team` |

### 3.3 例

```csv
チーム名,学籍番号,氏名
チームA,22001,山田太郎
チームA,22002,佐藤花子
チームA,22003,鈴木一郎
チームB,22004,田中次郎
チームB,22005,高橋三郎
```

---

## 4. 未提出判定への利用

`class-report-check` の Step 3 で履修者名簿が提供された場合、名簿に含まれる学生のうち提出物の対応表推定で検出されなかった学生を「未提出」として記録する。名簿が提供されない場合は提出物から検出された学生のみを採点対象とし、未提出判定は行わない。

---

## 5. class-report での扱い

`class-report` の Step 2（提出設定）でチームレポートが選択された場合、チーム構成情報を以下のいずれかで収集する:

- チーム構成CSVファイルのパスを受理（本スキーマに準拠）
- プレーンテキストでチーム名とメンバー一覧を直接入力

CSVが指定された場合は本スキーマで読み取り、構造化情報として後続ステップで利用する。直接入力された場合も、内部的には本スキーマと同じ構造に正規化して保持する。

`class-report` がチーム構成を保存する必要がある場合（例: 課題MDに含める、別ファイルとして書き出す）は、本スキーマに準拠したCSV形式で保存し、`class-report-check` がそのまま読めるようにする。
