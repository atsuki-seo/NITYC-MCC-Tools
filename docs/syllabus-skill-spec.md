# シラバス作成スキル仕様書

## 概要

高専のシラバス作成を効率化するClaude Codeスキルの仕様定義。エクセルファイル形式のシラバスに対し、ユーザー入力項目とClaude生成項目を分離し、MCCとの対応付けも行う。

---

## 1. 入力仕様

### 1a. ユーザー入力（エクセルから読み取る項目）

| シート | 項目 | 用途 |
|--------|------|------|
| **科目情報・到達目標** | 教科名、科目番号、科目区分 | 基本情報 |
| | 授業形態、単位の種別と単位数 | 基本情報 |
| | 開設学科、対象学年、開講期、週時間数 | スケジュール情報 |
| | 教科書/教材 | 参考情報 |
| | 担当教員 | 基本情報 |
| | 到達目標（複数行） | 授業計画生成の核 |
| | 学科の到達目標項目との関係 | MCC対応の参考 |
| **ルーブリック** | 評価観点（複数行） | 評価割合生成の核 |
| | 理想的/標準的/未到達レベルの目安 | 授業計画・評価の参考 |
| **教育方法等** | 授業の進め方・方法 | 概要・注意点生成の参考 |

### 1b. 対話で取得する項目

| 項目 | 選択肢 | 用途 |
|------|--------|------|
| 授業回数 | 数値（1〜32） | 授業計画の週数決定 |
| 定期試験の実施 | 中間のみ/期末のみ/両方/なし | 試験週の設定 |
| MCC対応の使用 | はい/いいえ | MCC参照の有無 |

---

## 2. 出力仕様

### 2a. 「教育方法等・自由使用枠」シート

| 項目 | 生成ロジック |
|------|-------------|
| **概要** | 到達目標を要約し、科目の目的・扱う内容・期待される成果を記述 |
| **注意点** | 授業の進め方から予習復習の推奨事項、評価上の注意を抽出・生成 |

### 2b. 「授業計画」シート

| 項目 | 生成ロジック |
|------|-------------|
| 週 | 1〜N週（ユーザー指定） |
| 授業内容 | 到達目標とルーブリックを分解し、週ごとに配分 |
| 週ごとの到達目標 | 各週で達成すべき具体的な目標を記述 |
| 試験週 | 定期試験実施の場合、8週目/16週目等に「中間試験」「期末試験」を設定 |

### 2c. 「評価割合」シート

| 項目 | 生成ロジック |
|------|-------------|
| 評価方法 | 定期試験、小テスト、レポート、口頭発表、成果物実技、ポートフォリオ、その他 |
| 配分 | ルーブリックの評価観点から推定（合計100%） |
| 観点との対応 | 知識の基本的な理解/汎用的技能/主体的・継続的な学習意欲/態度・指向性 |

### 2d. 「授業の属性・履修上の区分」シート

| 項目 | 生成ロジック |
|------|-------------|
| アクティブラーニング | 授業方法に「反転授業」「グループワーク」「演習」等が含まれるか |
| ICT利用 | 授業方法に「LMS」「オンライン」「プログラミング環境」等が含まれるか |
| 遠隔授業対応 | 授業方法に「オンライン対応」等が含まれるか |
| 実務経験のある教員 | ユーザー入力または別途確認 |

## 3. MCC対応仕様

### 3a. 参照データ

- `03-Resources/Repos/Kosen-MCC2023-Tech-IT/README.md`

### 3b. 対応付けルール

| MCCカテゴリ | キーワード例 |
|-------------|-------------|
| V-D-1 プログラミング | 変数、データ型、制御構造、サブルーチン、コンパイル |
| V-D-2 ソフトウェア | アルゴリズム、ソート、探索、データ構造、計算量 |
| V-D-3 コンピュータアーキテクチャ | 機械命令、論理回路、CPU |
| V-D-6 情報通信ネットワーク | TCP/IP、プロトコル、サーバ |
| V-D-8 その他 | データベース、SQL、セキュリティ |
| MCC plus 2023 | AI、機械学習、IoT、ROS |

### 3c. 出力

- 到達目標とMCCの対応表
- カバー推奨項目の提示
- 授業計画へのMCC項目の反映

---

## 4. 処理フロー仕様

```
Step 1: ファイル読み込み・検証
Step 2: 入力データの抽出・表示・確認
Step 3: 授業回数・定期試験の確認
Step 4: MCC対応付け分析（オプション）
Step 5: 出力項目の生成
Step 6: プレビュー・修正
Step 7: エクセル書き込み
Step 8: 完了報告
```

---

## 5. 技術仕様

### 5a. 依存関係

- `openpyxl` パッケージ（Pythonでのエクセル操作）

### 5b. エクセル構造（6シート構成）

1. 科目情報・到達目標
2. ルーブリック
3. 教育方法等・自由使用枠
4. 授業計画
5. 評価割合
6. 授業の属性・履修上の区分

### 5c. エラーハンドリング

| エラー | 対応 |
|--------|------|
| ファイル不存在 | エラーメッセージを表示して終了 |
| シート不足 | 不足シート名を表示して終了 |
| 到達目標が空 | 警告・手動入力を求める |
| 書き込み失敗 | バックアップからの復元を提案 |

---

## 6. 追加検討項目

| 項目 | 検討内容 |
|------|----------|
| **前年度シラバスの参照** | 過去のシラバスを参照して継続性を保つか |
| **複数科目の一括処理** | 同一教員の複数科目を連続処理するか |
| **テンプレートの柔軟性** | セル位置が異なる場合の対応（設定ファイル化など） |
| **出力の校正機能** | 生成後の誤字脱字チェック、文体統一 |
| **エクスポート形式** | エクセル以外（PDF、Markdown）への出力 |
